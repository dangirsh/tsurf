  <name>Task 2: Create modules/agentd.nix with option schema and jcard.toml rendering</name>
  <files>modules/agentd.nix</files>
  <action>
    Create `modules/agentd.nix` — the core NixOS module for agentd agent management.

    **Module structure:**

    1. **Option schema** (`services.agentd.agents`): An attrsOf submodule where each key is an agent name. Submodule options:
       - `enable` (bool, default true) — enable this agent instance
       - `package` (package, default pkgs.agentd) — agentd package
       - `harness` (enum ["claude-code" "opencode" "gemini-cli" "custom"], default "custom") — agent harness type
       - `command` (nullOr str, default null) — binary path for custom harness (required when harness=custom)
       - `agentBinary` (str, default "claude") — the agent binary name inside the sandbox (claude, codex, opencode, gemini, pi)
       - `prompt` (str, default "") — inline startup prompt (empty = interactive)
       - `promptFile` (nullOr path, default null) — prompt file path (overrides prompt)
       - `workdir` (str, default "/data/projects") — agent working directory
       - `restart` (enum ["no" "on-failure" "always"], default "no") — restart policy
       - `maxRestarts` (int, default 0) — max restart attempts (0 = unlimited)
       - `timeout` (str, default "") — Go duration format, empty = no timeout
       - `gracePeriod` (str, default "30s") — SIGINT grace period
       - `session` (str, default "") — tmux session name (defaults to agent name if empty)
       - `env` (attrsOf str, default {}) — additional environment variables
       - `environmentFile` (nullOr str, default null) — systemd EnvironmentFile path (for sops templates)
       - `extraArgs` (listOf str, default []) — extra CLI args for agentd
       - `sandbox` (bool, default true) — whether to run in bubblewrap sandbox
       - `user` (str) — the user identity inside the sandbox (for bwrap --uid, home dir paths). In public module defaults to "myuser"; private overlay overrides to "dangirsh".
       - `apiProxyPort` (nullOr int, default null) — if set, creates a socat TCP-to-Unix proxy on this port for the agent's API socket

    2. **jcard.toml rendering**: For each enabled agent, render `/etc/agentd/<name>/jcard.toml`:
       ```toml
       [agent]
       type = "native"
       harness = "<harness>"
       prompt = "<prompt>"
       workdir = "<workdir>"
       restart = "<restart>"
       max_restarts = <maxRestarts>
       grace_period = "<gracePeriod>"
       session = "<name>"

       [agent.env]
       <key> = "<value>"
       ```

       Use `environment.etc."agentd/${name}/jcard.toml".text` for rendering.

       If `harness = "custom"` and `command` is not null, there is a potential gap: the `custom` harness in agentd defaults its binary to "agent". The jcard.toml does not have a `binary` field for custom harness (based on research). The wrapper script must be named `agent` on PATH, OR agentd must be forked to support a configurable binary path.

       **Resolution:** The custom harness looks for `agent` on PATH. Use `pkgs.writeShellScriptBin "agent" ''...bwrap wrapper...''` which produces a proper `$out/bin/agent` structure. Add this derivation to the systemd service `path`. The custom harness finds `agent` on PATH and executes it. No conditional branching — this is the definitive approach.

    3. **Bwrap wrapper generation**: For each agent with `sandbox = true`, generate a bwrap wrapper script (Nix derivation) that reproduces the existing sandbox policy from `agent-compute.nix`. The wrapper:
       - Reads API keys from environment (inherited from agentd's systemd EnvironmentFile, not from /run/secrets/ files directly)
       - Sets up all the same bind mounts as current agent-spawn:
         - ro-bind: /nix/store, /run/current-system, /etc (selected files), /data/projects (all siblings)
         - bind (rw): project dir, containers dirs
         - hidden: /run/secrets, ~/.ssh, /var/run/docker.sock
       - Applies same namespace isolation: unshare-user, unshare-ipc, unshare-uts, unshare-pid, unshare-cgroup, disable-userns
       - Sets sandbox env vars: HOME, USER, SHELL, TERM, LANG, PATH (with sandbox-docker-compat), SANDBOX=1, SANDBOX_NAME, SANDBOX_PROJECT, NIX_REMOTE=daemon
       - Passes API keys via --setenv from inherited env vars (ANTHROPIC_API_KEY, OPENAI_API_KEY, GITHUB_TOKEN, etc.)
       - Ends with: `exec bwrap <args> -- <agentBinary> "$@"`

       The wrapper is a `pkgs.writeShellScriptBin "agent"` derivation (produces `$out/bin/agent`). The env vars come from the parent process (agentd's tmux session inherits the systemd service environment).

       **Key difference from agent-spawn:** agent-spawn reads secrets from /run/secrets/ files. The agentd wrapper receives them as env vars (from EnvironmentFile -> systemd -> agentd -> tmux -> bwrap inherits). The bwrap wrapper uses `--setenv ANTHROPIC_API_KEY "$ANTHROPIC_API_KEY"` (reading from process env, not file).

       For agents with `sandbox = false`, the wrapper simply execs the agent binary directly (no bwrap).

    4. **Audit logging**: Preserve the journald audit logging from agent-spawn. Each bwrap wrapper logs spawn events via `systemd-cat -t agentd-spawn`.

    **Module-level let bindings:**
    - `zmx` package (from packages/zmx.nix) -- kept available on system PATH for human debugging
    - `sandbox-docker-compat` derivation (podman -> docker symlink for sandbox PATH)
    - `mkAgentWrapper` function that takes agent name + config and returns a `pkgs.writeShellScriptBin "agent" ''...''` derivation (for custom harness PATH lookup)

    **Important:** The `user` field controls paths inside the sandbox (home dir, profile path, etc.). The public module uses `myuser` as default. The private overlay overrides this to `dangirsh`. The bwrap wrapper uses `cfg.user` for:
    - `--dir /home/${cfg.user}`
    - `--setenv HOME /home/${cfg.user}`
    - `--setenv USER ${cfg.user}`
    - `--ro-bind-try /etc/profiles/per-user/${cfg.user} ...`
    - `--ro-bind-try /home/${cfg.user}/.claude ...`
    - PATH includes `/etc/profiles/per-user/${cfg.user}/bin`

    **Assertions:**
    - Assert that `command` is non-null when `harness = "custom"` and `sandbox = false`
    - Assert that `apiProxyPort` values are unique across all agents
    - NOTE: No `workdir` path assertion — the convention is `/data/projects` but this is not a security property and would be overly restrictive for non-standard layouts
  </action>

<task type="auto">
  <name>Task 3: Generate systemd services and socat API proxies</name>
  <files>modules/agentd.nix</files>
  <action>
    This task is part of the same `modules/agentd.nix` file from Task 2 — it covers the systemd service generation and API proxy aspects.

    **Systemd service per agent** (`systemd.services."agentd-${name}"`):

    ```nix
    systemd.services."agentd-${name}" = {
      description = "agentd agent: ${name}";
      after = [ "network-online.target" "sops-nix.service" ];
      wants = [ "network-online.target" ];
      wantedBy = [ "multi-user.target" ];

      path = with pkgs; [ tmux sudo bubblewrap coreutils ];

      serviceConfig = {
        ExecStart = lib.concatStringsSep " " ([
          "${cfg.package}/bin/agentd"
          "-config /etc/agentd/${name}/jcard.toml"
          "-api-socket /run/agentd/${name}/agentd.sock"
          "-tmux-socket /run/agentd/${name}/tmux.sock"
          "-secret-dir /run/agentd/${name}/secrets/"
          # NOTE: the -agent-user flag is passed via extraArgs from the private overlay
          # e.g., extraArgs = ["-agent-user" "dangirsh"]
          # This controls the tmux runAs user (separate from the bwrap sandbox user)
        ] ++ cfg.extraArgs);

        RuntimeDirectory = "agentd/${name}";
        RuntimeDirectoryMode = "0750";
        DynamicUser = false;
        User = "root";
        Restart = "on-failure";
        RestartSec = 5;
      } // lib.optionalAttrs (cfg.environmentFile != null) {
        EnvironmentFile = cfg.environmentFile;
      };
    };
    ```

    **Critical:** The per-agent wrapper derivation (named `agent`) must be on the service PATH so agentd's custom harness finds it. Add it to the `path` list:
    ```nix
    path = [ (mkAgentWrapper name cfg) ] ++ (with pkgs; [ tmux sudo bubblewrap coreutils ]);
    ```

    **socat API proxy** (for agents with `apiProxyPort != null`):

    ```nix
    systemd.services."agentd-proxy-${name}" = lib.mkIf (cfg.apiProxyPort != null) {
      description = "TCP proxy for agentd-${name} API";
      after = [ "agentd-${name}.service" ];
      bindsTo = [ "agentd-${name}.service" ];
      wantedBy = [ "multi-user.target" ];
      serviceConfig = {
        ExecStart = "${pkgs.socat}/bin/socat TCP-LISTEN:${toString cfg.apiProxyPort},fork,reuseaddr UNIX-CONNECT:/run/agentd/${name}/agentd.sock";
        Restart = "on-failure";
        RestartSec = 3;
      };
    };
    ```

    The `bindsTo` ensures the proxy stops when the agent stops.

    **sops.templates for agent secrets:** Create a MINIMAL shared sops template referencing ONLY `anthropic-api-key` (guaranteed to exist in both `secrets/neurosys.yaml` and `secrets/ovh.yaml`). Host-specific secrets (openrouter, xai, google, etc.) are added via per-agent `environmentFile` overrides or the per-agent `env` option in the private overlay.

    ```nix
    # In the module config section (not per-agent):
    sops.templates."agentd-env" = {
      content = ''
        ANTHROPIC_API_KEY=${config.sops.placeholder."anthropic-api-key"}
      '';
      owner = "root";
    };
    ```

    Each agent's `environmentFile` defaults to this template's path:
    ```nix
    environmentFile = lib.mkDefault config.sops.templates."agentd-env".path;
    ```

    **Per-host secret extension pattern:** The private overlay creates host-specific sops templates that include additional secrets (openrouter, xai, github-pat, etc.) and overrides `environmentFile` for agents that need them. For example, `sops.templates."agentd-env-full"` in the private overlay references all available secrets for that host. Individual agents like claw-swap get a custom template with `ANTHROPIC_BASE_URL` pointing to the secret proxy.

    This design avoids `nix flake check` failures when a secret referenced in a sops placeholder does not exist in a host's sops file.

    **NOTE:** The sops.templates declaration should only be created if there are any enabled agents. Guard with `lib.mkIf (enabledAgents != {})`.
  </action>
