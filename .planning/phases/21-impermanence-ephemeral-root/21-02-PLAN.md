The human operator must execute the following steps. This is a **destructive operation** — the server's disk will be completely reformatted from ext4 to BTRFS.

**Step 1: Verify backup is current (on LOCAL machine)**

```bash
# Set restic credentials (extract from sops)
cd /path/to/neurosys
export AWS_ACCESS_KEY_ID="$(sops -d secrets/neurosys.yaml | grep b2-account-id | cut -d: -f2 | tr -d ' ')"
export AWS_SECRET_ACCESS_KEY="$(sops -d secrets/neurosys.yaml | grep b2-account-key | cut -d: -f2 | tr -d ' ')"
export RESTIC_REPOSITORY="s3:s3.eu-central-003.backblazeb2.com/SyncBkp"

# Verify repository and check latest snapshot is < 24h old
restic check
restic snapshots --latest 1
```

If the latest snapshot is stale, trigger a fresh backup on the live server BEFORE proceeding:
```bash
ssh root@neurosys systemctl start restic-backups-b2.service
# Wait for completion:
ssh root@neurosys journalctl -u restic-backups-b2.service -f
```

**Step 2: Extract SSH host key from backup into /persist layout (on LOCAL machine)**

The key MUST go to `/persist/etc/ssh/` (not `/etc/ssh/`). The initrd rollback script wipes `@root` on the FIRST boot before sops-nix runs. By placing the key under `/persist/`, it survives the wipe and the impermanence bind-mount makes it visible at `/etc/ssh/ssh_host_ed25519_key` for sops-nix.

```bash
mkdir -p /tmp/host-keys/persist/etc/ssh
restic restore latest --target /tmp/host-key-restore \
  --include /etc/ssh/ssh_host_ed25519_key \
  --include /etc/ssh/ssh_host_ed25519_key.pub

# Move from restore layout into /persist layout for --extra-files
cp /tmp/host-key-restore/etc/ssh/ssh_host_ed25519_key /tmp/host-keys/persist/etc/ssh/
cp /tmp/host-key-restore/etc/ssh/ssh_host_ed25519_key.pub /tmp/host-keys/persist/etc/ssh/
chmod 600 /tmp/host-keys/persist/etc/ssh/ssh_host_ed25519_key
chmod 644 /tmp/host-keys/persist/etc/ssh/ssh_host_ed25519_key.pub
rm -rf /tmp/host-key-restore

# Verify — key is at the /persist path, NOT /etc/ssh
ls -la /tmp/host-keys/persist/etc/ssh/
```

**Step 3: Run nixos-anywhere (DESTRUCTIVE — wipes disk)**

```bash
cd /path/to/neurosys

# nixos-anywhere uses the PUBLIC IP directly (not Tailscale)
# because after disk wipe, Tailscale won't be running until first boot
nixos-anywhere --extra-files /tmp/host-keys \
  --flake .#neurosys root@161.97.74.121
```

This will:
1. kexec into installer environment
2. Partition disk with BTRFS via disko (5 subvolumes)
3. Install NixOS with impermanence config
4. Inject SSH host key from --extra-files to `/persist/etc/ssh/` (survives initrd root wipe; impermanence bind-mount exposes it at `/etc/ssh/` for sops-nix)
5. Reboot — initrd wipes `@root`, but `/persist/etc/ssh/` is on `@persist` subvolume and survives

**Step 5: Restore stateful data from restic**

```bash
ssh root@161.97.74.121
```

On the server:

```bash
# Source restic credentials from sops secrets
source /run/secrets/rendered/restic-b2-env
export RESTIC_PASSWORD=$(cat /run/secrets/restic-password)
export RESTIC_REPOSITORY="s3:s3.eu-central-003.backblazeb2.com/SyncBkp"

# Verify repository
restic check
restic snapshots --latest 1

# Stop services before restore
systemctl stop docker.service
systemctl stop home-assistant.service
systemctl stop tailscaled.service
systemctl stop syncthing.service
systemctl stop prometheus.service

# Restore to a staging directory first
mkdir -p /tmp/restore
restic restore latest --target /tmp/restore

# The backup was from the old / layout. Copy to /persist layout:
# SSH keys (already injected but restore ensures latest)
cp -a /tmp/restore/etc/ssh/* /persist/etc/ssh/
cp -a /tmp/restore/etc/machine-id /persist/etc/machine-id

# Service state
cp -a /tmp/restore/var/lib/tailscale /persist/var/lib/tailscale
cp -a /tmp/restore/var/lib/claw-swap /persist/var/lib/claw-swap
cp -a /tmp/restore/var/lib/parts /persist/var/lib/parts
cp -a /tmp/restore/var/lib/hass /persist/var/lib/hass
cp -a /tmp/restore/var/lib/prometheus2 /persist/var/lib/prometheus2
cp -a /tmp/restore/var/lib/prometheus-node-exporter /persist/var/lib/prometheus-node-exporter
cp -a /tmp/restore/var/lib/fail2ban /persist/var/lib/fail2ban
cp -a /tmp/restore/var/lib/nixos /persist/var/lib/nixos
mkdir -p /persist/var/lib/private
cp -a /tmp/restore/var/lib/private/esphome /persist/var/lib/private/esphome 2>/dev/null || true

# Systemd state
mkdir -p /persist/var/lib/systemd
cp -a /tmp/restore/var/lib/systemd/timers /persist/var/lib/systemd/timers
cp -a /tmp/restore/var/lib/systemd/timesync /persist/var/lib/systemd/timesync
cp -a /tmp/restore/var/lib/systemd/linger /persist/var/lib/systemd/linger 2>/dev/null || true
cp -a /tmp/restore/var/lib/systemd/random-seed /persist/var/lib/systemd/random-seed 2>/dev/null || true
mkdir -p /persist/var/lib/systemd/coredump

# User data
cp -a /tmp/restore/home/dangirsh /persist/home/dangirsh
cp -a /tmp/restore/root /persist/root

# Code repos
mkdir -p /persist/data
cp -a /tmp/restore/data/projects /persist/data/projects

# Docker data goes to the @docker subvolume (already mounted at /var/lib/docker)
# Only restore non-overlay2 data — images are rebuilt by NixOS activation
cp -a /tmp/restore/var/lib/docker/containers /var/lib/docker/containers 2>/dev/null || true
cp -a /tmp/restore/var/lib/docker/network /var/lib/docker/network 2>/dev/null || true
cp -a /tmp/restore/var/lib/docker/volumes /var/lib/docker/volumes 2>/dev/null || true

# Fix ownership
chown -R dangirsh:users /persist/home/dangirsh/
chown -R dangirsh:users /persist/data/projects/

# Clean up staging
rm -rf /tmp/restore

# Reboot to activate impermanence bind-mounts
reboot
```
