  <action>
    Create `/data/projects/neurosys/scripts/bootstrap-contabo.sh` adapting the OVH bootstrap script with these key differences:

    **Contabo-specific changes from OVH:**
    1. VPS_IP = `161.97.74.121` (Contabo, not OVH)
    2. SSH auth: Use `sshpass -p 'fuckingcontabosecurityshit'` for password-based root SSH (Contabo provides root password after Ubuntu reinstall, not deploy key)
    3. No deploy key generation (Contabo uses password, not key auth)
    4. No PAM password expiry handling (no ubuntu@ user, Contabo gives root directly)
    5. No pexpect dependency
    6. EXTRA_FILES points to `tmp/neurosys-host-keys` (not `tmp/ovh-host-keys`)
    7. FLAKE_TARGET = `/data/projects/private-neurosys#neurosys` (private overlay, Contabo config)
    8. TAILSCALE_HOSTNAME = `neurosys` (not `neurosys-prod`)
    9. nixos-anywhere needs `--env-password` flag to pass SSH password via SSHPASS env var

    **Script structure:**
    - Phase 0: Verify preconditions (sshpass installed, SSH host key exists in tmp/, tailscale CLI available)
    - Phase 1: Clear old SSH known_hosts entry, poll for SSH access using sshpass
    - Phase 2: Run nixos-anywhere with --extra-files and password auth
    - Phase 3: Wait for Tailscale hostname to appear, print success

    **Critical: nixos-anywhere password auth:**
    nixos-anywhere can use `--env-password` to read from `SSHPASS` env var. Set `SSHPASS=fuckingcontabosecurityshit` before the command and pass `--env-password` flag.

    If `--env-password` is not available in the nixos-anywhere version, fall back to:
    1. First copy a temporary SSH key to root@IP using sshpass
    2. Then run nixos-anywhere with `-i` pointing to that temporary key

    Use the fallback approach (copy key then use it) as it is more reliable:
    ```bash
    # Generate temp deploy key
    ssh-keygen -t ed25519 -f "$DEPLOY_KEY" -N "" -C "contabo-bootstrap"
    # Copy it using sshpass
    sshpass -p "$CONTABO_PASS" ssh-copy-id -i "$DEPLOY_KEY.pub" -o StrictHostKeyChecking=no root@$VPS_IP
    # Then nixos-anywhere uses the key
    nix run github:nix-community/nixos-anywhere -- --extra-files "$EXTRA_FILES" --flake "$FLAKE_TARGET" -i "$DEPLOY_KEY" "root@${VPS_IP}"
    ```

    Make the script executable: `chmod +x scripts/bootstrap-contabo.sh`
  </action>
