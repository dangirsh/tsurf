<task type="human">
  <name>Task 1: Pre-bootstrap Tailscale cleanup</name>
  <files></files>
  <action>
    Before bootstrapping, remove the old 'neurosys' node from the Tailscale admin console to prevent the new node from getting a '-1' suffix.

    1. Go to https://login.tailscale.com/admin/machines
    2. Find the old 'neurosys' node (it should be offline/expired)
    3. Delete it
    4. Verify it's gone: `tailscale status | grep neurosys` should show nothing

    If the old node was already removed, skip this step.

    Also generate a new reusable Tailscale auth key:
    1. Go to https://login.tailscale.com/admin/settings/keys
    2. Create a new auth key (reusable, no expiry or long expiry)
    3. Update the tailscale-authkey secret in the private overlay's secrets/neurosys.yaml:
       ```bash
       cd /data/projects/private-neurosys
       sops --set '["tailscale-authkey"] "tskey-auth-XXXXX"' secrets/neurosys.yaml
       git add secrets/neurosys.yaml
       git commit -m "chore(46): update Tailscale auth key for Contabo re-bootstrap"
       git push
       ```
  </action>
  <verify>tailscale status 2>/dev/null | grep -v '\-1' | grep neurosys || echo "No stale neurosys node (good)"</verify>
  <done>Old Tailscale node removed and new auth key in sops</done>
</task>

<task type="human">
  <name>Task 3: Run bootstrap-contabo.sh</name>
  <files></files>
  <action>
    Run the bootstrap script. This will take 10-20 minutes.

    ```bash
    cd /data/projects/neurosys
    ./scripts/bootstrap-contabo.sh
    ```

    The script will:
    1. Verify preconditions (sshpass, SSH host key, tailscale CLI)
    2. Test SSH connectivity to root@161.97.74.121 using sshpass
    3. Run nixos-anywhere (wipes Ubuntu, installs NixOS, reboots)
    4. Wait for 'neurosys' to appear in Tailscale

    Monitor the output for errors. Common failure modes:
    - **SSH auth failure:** Verify the Contabo password is correct
    - **nixos-anywhere build failure:** Check for Nix evaluation errors (missing inputs, syntax issues)
    - **Tailscale never appears:** sops secrets may have failed to decrypt (wrong age key)

    If nixos-anywhere fails:
    - The VPS may be in an unbootable state
    - User can reinstall Ubuntu from Contabo panel and retry
    - Check the nixos-anywhere output for specific errors
  </action>
  <verify>tailscale status 2>/dev/null | grep -q 'neurosys' && echo "Bootstrap OK" || echo "Bootstrap may have failed"</verify>
  <done>nixos-anywhere complete, Contabo booting NixOS with Tailscale connected</done>
</task>

<task type="human">
  <name>Task 4: Verify basic boot and services</name>
  <files></files>
  <action>
    After Tailscale shows 'neurosys', verify basic functionality:

    ```bash
    # SSH via Tailscale
    ssh root@neurosys 'hostname && uname -a'
    # Expected: neurosys, Linux ... NixOS

    # Check sops secrets decrypted
    ssh root@neurosys 'ls /run/secrets/ | head -20'
    # Expected: anthropic-api-key, github-pat, tailscale-authkey, etc.

    # Check impermanence mounts
    ssh root@neurosys 'mount | grep /persist'
    # Expected: BTRFS subvolume mounts

    # Check basic services
    ssh root@neurosys 'systemctl is-active tailscaled sshd fail2ban'
    # Expected: active, active, active

    # Check Docker
    ssh root@neurosys 'systemctl is-active docker && docker ps'
    # Expected: active, containers starting

    # Check Prometheus
    ssh root@neurosys 'curl -s http://localhost:9090/-/healthy'
    # Expected: Prometheus Server is Healthy.
    ```

    If any services are failing, check journal:
    ```bash
    ssh root@neurosys 'journalctl -b --no-pager -p err | head -50'
    ```
  </action>
  <verify>ssh root@neurosys 'systemctl is-active tailscaled sshd fail2ban' && echo "Basic boot OK"</verify>
  <done>Contabo VPS booted with NixOS, basic services verified</done>
</task>
