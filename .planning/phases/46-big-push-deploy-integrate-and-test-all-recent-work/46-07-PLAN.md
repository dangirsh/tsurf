  <action>
    Deploy the updated config to OVH:

    ```bash

    ./scripts/deploy.sh --node ovh
    ```

    This will build (remotely on OVH) and activate the new config including:
    - matrix-conduit (Conduit homeserver)
    - mautrix-telegram
    - mautrix-whatsapp
    - mautrix-signal

    **Expected build notes:**
    - First build with matrix modules will be slower (new packages: conduit, mautrix-*)
    - olm-3.2.16 will require `permittedInsecurePackages` (already in matrix.nix)
    - If the build fails on olm version mismatch, check the actual olm version in nixpkgs 25.11

    - This means OVH lost connectivity during activation
    - Check if matrix services caused a port conflict or networking issue

    **If build fails:**
    - Common issue: sops secrets can't decrypt (wrong key or missing secrets)
    - Check: `ssh root@neurosys-prod journalctl -u sops-nix --since '5 min ago'`
    - Common issue: olm version changed in nixpkgs
    - Check: `nix eval .#nixosConfigurations.ovh.config.nixpkgs.config.permittedInsecurePackages`
  </action>
