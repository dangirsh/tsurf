<task type="auto">
  <name>Task 2: Create deploy script with both modes and health verification</name>
  <files>scripts/deploy.sh</files>
  <action>
Create `scripts/deploy.sh` as an executable bash script. The script implements two deploy modes selected by a `--mode` flag (defaulting to `local`):

**Script structure:**

```
#!/usr/bin/env bash
set -euo pipefail
```

**Variables:**
- `FLAKE_DIR`: Resolved to the repo root (parent of scripts/)
- `TARGET`: `root@acfs` (SSH via Tailscale)
- `MODE`: `local` (default) or `remote` (set via `--mode remote`)
- `SECONDS=0`: bash built-in timer
- `CONTAINERS`: Array of ALL expected containers: `parts-tools`, `parts-agent`, `claw-swap-db`, `claw-swap-app`, `claw-swap-caddy`

**Argument parsing:**
- `--mode local` (default): local-push mode
- `--mode remote`: remote-self-deploy mode
- `--skip-update`: Skip the `nix flake update parts` step (useful when deploying neurosys changes without updating parts)
- `--help`: Print usage

**Local-push mode flow:**
1. Print "Updating parts flake input..." (unless `--skip-update`)
2. Run `nix flake update parts --flake "$FLAKE_DIR"` (unless `--skip-update`)
3. Extract parts revision: `nix flake metadata "$FLAKE_DIR" --json | jq -r '.locks.nodes.parts.locked.rev'`
4. Print "Building and deploying to $TARGET..."
5. Run: `nix shell nixpkgs#nixos-rebuild -c nixos-rebuild switch --flake "$FLAKE_DIR#acfs" --target-host "$TARGET" --build-host localhost`
6. Proceed to health verification

**Remote-self-deploy mode flow:**
1. SSH into server and run a heredoc script:
   - `cd /data/projects/neurosys`
   - `git pull --ff-only`
   - `nix flake update parts` (unless `--skip-update`)
   - `nixos-rebuild switch --flake .#acfs`
2. Extract parts revision from local flake.lock (for reporting)
3. Proceed to health verification

**Health verification (shared by both modes):**
1. Poll every 2 seconds, up to 15 attempts (30 seconds total)
2. For each container in CONTAINERS array:
   - Run `ssh "$TARGET" "docker ps --filter name=^${c}\$ --filter status=running -q"`
   - If output is non-empty, container is running
3. Track which containers are running and which are not
4. Break early if all containers are running

**Success output:**
```
=== Deploy SUCCESS ===
Parts revision: <rev>
Duration: Xm Ys
Container status:
  <docker ps table for all containers>

NOTE: flake.lock was updated. Remember to commit when ready:
  git add flake.lock && git commit -m "chore: update parts input to <short-rev>"
```

**Failure output:**
```
=== Deploy FAILED ===
Containers not running after 30s:
  <list of failed containers>
All container status:
  <docker ps -a table>

To rollback:
  ssh root@acfs nixos-rebuild switch --rollback
```

Make the script executable: `chmod +x scripts/deploy.sh`

**Important implementation details:**
- Use `set -euo pipefail` at the top
- The `FLAKE_DIR` must be computed as the absolute path to the repo root: `FLAKE_DIR="$(cd "$(dirname "$0")/.." && pwd)"`
- Docker ps filter uses `name=^container-name$` with anchored regex to avoid partial matches
- The polling loop should NOT use `set -e` failure on the ssh commands (use explicit if/grep checks)
- The `--skip-update` flag should be a simple boolean variable checked before the update step
- Do NOT auto-commit flake.lock -- just print the reminder
