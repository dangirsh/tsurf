```bash

```

**Step 2: Deploy to OVH**

Use the deploy script to push the new configuration:
```bash
./scripts/deploy.sh --node ovh
```

This activates the nginx module on OVH. After activation:
- nginx will start and bind ports 80/443
- ACME will attempt to get certificates for all configured domains
- claw-swap Docker Caddy should already be removed (from claw-swap flake update)

**Step 3: Verify nginx is running**

```bash
ssh root@neurosys-prod "systemctl status nginx; systemctl status acme-dangirsh.org"
```

**Step 4: Test claw-swap.com first (existing DNS)**

claw-swap.com DNS should already point to OVH (135.125.196.143). Verify it works through nginx:
```bash

```

If claw-swap.com ACME fails (DNS not yet pointing here), the certificate won't issue. Check ACME logs:
```bash
ssh root@neurosys-prod "journalctl -u acme-claw-swap.com --no-pager -n 50"
```

This should return the site HTML (without TLS, since ACME won't have a cert yet for a domain that doesn't point here).

**Human action required: Update DNS records for dangirsh.org**

This is an irreversible operation (until DNS propagation completes, which takes 1-48 hours depending on TTL).

**Pre-cutover checklist:**
1. Lower TTL on existing dangirsh.org A record to 300 seconds (5 minutes), if not already done 24-48 hours ago
2. Verify the site content on OVH is correct: `curl -H "Host: dangirsh.org" http://135.125.196.143/`
3. Note current NFS IP for rollback: 208.94.117.137

**DNS changes to make (Google Domains or current registrar):**

| Record | Type | Old Value | New Value |
|--------|------|-----------|-----------|
| dangirsh.org | A | 208.94.117.137 (NFS) | 135.125.196.143 (OVH) |
| www.dangirsh.org | A or CNAME | (current value) | 135.125.196.143 (OVH) |

**After DNS change:**

Wait for propagation. Check with:
```bash
dig +short dangirsh.org
# Should return 135.125.196.143
```

Once DNS resolves to OVH, ACME will be able to issue certificates via HTTP-01 challenge. Force ACME renewal:
```bash
ssh root@neurosys-prod "systemctl start acme-dangirsh.org; systemctl start acme-www.dangirsh.org"
```

Then verify TLS works:
```bash
curl -I https://dangirsh.org
curl -I https://www.dangirsh.org
```

The www redirect should return a 301 to https://dangirsh.org.

**Resume signal:** Type "dns-done" after confirming DNS propagation and ACME cert issuance, or describe issues.

**Comprehensive verification after DNS propagation:**

1. **dangirsh.org content check:**
   ```bash
   curl -s https://dangirsh.org | grep -o '<title>[^<]*</title>'
   curl -s https://dangirsh.org/about.html | head -5
   curl -s -o /dev/null -w "%{http_code}" https://dangirsh.org/posts/
   ```

2. **www redirect check:**
   ```bash
   curl -sI https://www.dangirsh.org | grep -E '^(HTTP|Location)'
   # Expect: HTTP/2 301, Location: https://dangirsh.org/
   ```

3. **claw-swap.com check:**
   ```bash
   curl -s -o /dev/null -w "%{http_code}" https://claw-swap.com
   # Expect: 200
   ```

4. **TLS certificate check:**
   ```bash
   echo | openssl s_client -connect dangirsh.org:443 -servername dangirsh.org 2>/dev/null | openssl x509 -noout -issuer -dates
   # Expect: issuer includes "Let's Encrypt", dates are current
   ```

5. **Static asset caching check:**
   ```bash
   curl -sI https://dangirsh.org/css/default.css | grep -E '^(Cache-Control|Expires)'
   # Expect: Cache-Control with "public, immutable" and 30d expiry
   ```

6. **ACME renewal timer check:**
   ```bash
   ssh root@neurosys-prod "systemctl list-timers 'acme-*' --no-pager"
   # All ACME timers should be active and scheduled
   ```

7. **No stale Caddy references:**
   ```bash
   ssh root@neurosys-prod "docker ps -a | grep caddy || echo 'No caddy containers'"
   # Expect: No caddy containers
   ```

If any check fails, investigate before proceeding. The old NFS hosting is still available as fallback (flip DNS A record back to 208.94.117.137).
