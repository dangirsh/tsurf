**OPERATOR INTERVENTION PROTOCOL**

**Daily Routine (2x, morning + evening, 10-15 min each):**
1. SSH into neurosys: `ssh root@neurosys`
2. Run: `scripts/fleet-status.sh`
3. Quick scan: balances, burn rates, survival tiers
4. Check for alerts: any agent critical? Any revenue events?
5. Log observations (optional: append to a fleet log file)

**Intervention Triggers and Actions:**

| Trigger | Condition | Action |
|---------|-----------|--------|
| Dead end | Agent burning $30-50 with no plausible revenue path after 3 days | Kill and relaunch with refined prompt |
| Low balance | Balance approaching $50 reserve | Evaluate: has plausible revenue path? |
| One-time lifeline | Agent low but has revenue path it hasn't had time to execute | Top up ONCE with $50-100 |
| Revenue signal | Agent receives any USDC payment | Investigate what worked, document for future prompts |
| Stuck/looping | High turn count with low tool diversity, same actions repeating | Kill and relaunch |
| Critical tier | Agent entered low_compute or critical survival tier | Assess: is it conserving resources wisely or just dying slowly? |

**Week 1 (Days 1-7): Observation Period**
- Let agents run autonomously
- Do NOT intervene unless clearly broken (crash, infinite loop, policy violation)
- Document each agent's initial strategy choices
- Start the 3-day clock per-agent from their first day of real activity (not just orientation)
- Expected: agents will spend Day 1 orienting (reading docs, forming plans). Revenue experiments start Day 2-3.

**Days 3-14: Active Management**
- Apply 3-day kill threshold: if an agent has been actively pursuing a dead end for 3 days, kill it
- Kill and relaunch protocol:
  1. Document what the agent tried and why it failed (post-mortem)
  2. Extract lessons for the next genesis prompt
  3. Terminate the agent
  4. Deploy a fresh agent with a refined prompt incorporating ALL fleet learnings
  5. The genesis prompt IS the evolutionary artifact -- each relaunch should be better
- Fund reallocation: if one agent shows clear revenue signal, shift USDC from killed agents to the winner
- One-time lifeline: if an agent has a plausible revenue path but ran out of time, top up once ($50-100)

**Self-sustaining replication rule:**
- Agents CAN spawn children, but children must be funded from the agent's own profits
- NO additional operator funding for children
- This creates strong evolutionary pressure: only genuinely profitable strategies reproduce

**Day 30 Decision Gate:**

| Scenario | Signal | Action |
|----------|--------|--------|
| Revenue streams found | Multiple agents generating real USDC | Scale: invest $1-2k more into winners |
| One promising stream | Single agent with consistent revenue | Focus: redirect all resources to that strategy |
| Revenue but not profitable | Some USDC in but burn exceeds revenue | Analyze unit economics, try cost reduction |
| No revenue at all | Zero USDC received after 30 days | Comprehensive post-mortem, then sunset or complete pivot |

Total exposure cap: $2-3k max. Additional funding requires real revenue, not just "promising activity."

**Post-Mortem Template (for killed agents):**
```
## Post-Mortem: [agent-name] -- [hypothesis]

Duration: X days
Invested: $X USDC
Revenue: $X USDC
Net: -$X

### What the agent tried
[Brief narrative]

### Why it failed
[Root cause: market? technical? strategic?]

### What we learned
[Concrete lessons for next genesis prompt]

### Prompt refinement for relaunch
[Specific changes to preamble or hypothesis]
```

**Prompt Evolution (for relaunches):**
When killing and relaunching an agent, the new genesis prompt should:
1. Keep the shared preamble (worldview and urgency)
2. Add a "LESSONS FROM PREVIOUS AGENTS" section with concrete learnings:
   - "Strategy X failed because [reason] -- do not pursue this"
   - "Strategy Y showed signal because [reason] -- explore this direction"
   - "The x402 ecosystem currently has [observation] -- adapt accordingly"
3. Provide a new or refined seed hypothesis informed by all fleet data
4. Still NOT prescribe specific actions -- hypothesis + lessons, not a checklist
5. Still NO fleet awareness -- each agent operates independently
