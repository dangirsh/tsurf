Create `nix/module.nix` — the NixOS module that parts' flake.nix exports as `nixosModules.default`.

The module function uses a curried pattern: the outer function receives `{ self, sops-nix, ... }` from flake.nix, the inner function receives `{ config, lib, pkgs, ... }` from the NixOS module system.

**Critical rules (from research + locked decisions):**
- Do NOT import `sops-nix.nixosModules.sops` — agent-neurosys already imports it. Just declare `sops.secrets` entries.
- Do NOT set `sops.defaultSopsFile` — agent-neurosys sets its own default. Use per-secret `sopsFile` overrides.
- Do NOT declare `virtualisation.docker.enable` — agent-neurosys owns Docker engine config.
- Do NOT declare firewall, Tailscale, SSH, boot, or user config.
- Use `self + "/secrets/parts.yaml"` for sopsFile (NOT relative paths — they resolve from the importing flake).

**Module must declare:**

   - google-oauth-client-id, google-oauth-client-secret, google-oauth-refresh-token
   - telegram-user-id, parts-psk, approval-api-token
   - Each with: `sopsFile = self + "/secrets/parts.yaml";`
   - Each with appropriate `restartUnits` — secrets used by parts-tools should restart `docker-parts-tools.service`, secrets used by parts-agent should restart `docker-parts-agent.service`
   - parts-psk and llm-proxy-token are used by BOTH containers — restart both services

<task type="auto">
  <name>Task 2: Add parts as flake input to agent-neurosys and verify integration</name>
  <files>/data/projects/agent-neurosys/flake.nix, /data/projects/agent-neurosys/flake.lock</files>
  <action>
Update `/data/projects/agent-neurosys/flake.nix` to import parts as a flake input.

1. **Add parts input:**
   In the `inputs` set, add:
   ```nix
   parts = {
     url = "path:/data/projects/parts";  # Local path for development; change to github:dangirsh/parts for production
     inputs.nixpkgs.follows = "nixpkgs";
     inputs.sops-nix.follows = "sops-nix";
   };
   ```
   Using `path:` for local development so changes to the parts repo are picked up immediately without pushing to GitHub. The `follows` directives ensure parts uses the same nixpkgs and sops-nix as agent-neurosys (preventing Pitfall 6: version mismatch).

2. **Add parts to outputs function args:**
   Update the outputs function to include `parts`:
   ```nix
   outputs = { self, nixpkgs, home-manager, sops-nix, disko, parts, ... } @ inputs: {
   ```

3. **Import parts module in acfs configuration:**
   Add `inputs.parts.nixosModules.default` to the modules list:
   ```nix
   modules = [
     disko.nixosModules.disko
     sops-nix.nixosModules.sops
     home-manager.nixosModules.home-manager
     inputs.parts.nixosModules.default   # <-- add this line
     ./hosts/acfs
     {
       home-manager.useGlobalPkgs = true;
       home-manager.useUserPackages = true;
       home-manager.extraSpecialArgs = { inherit inputs; };
       home-manager.users.dangirsh = import ./home;
     }
   ];
   ```

4. **Update flake.lock:**
   Run `nix flake lock --update-input parts` from the agent-neurosys directory to pin the parts flake.

5. **Verify integration:**
   Run `nix flake check` from the agent-neurosys directory. This evaluates the full NixOS configuration including the parts module. If it passes, the entire integration is valid.

   If `nix flake check` fails:
   - "option 'sops' already declared" = parts module is importing sops-nix (fix module.nix, remove the import)
   - "option 'virtualisation.docker' already declared" = parts module is enabling docker (fix module.nix)
   - "No such file or directory" for secrets = sopsFile path resolution issue (fix to use `self + "/secrets/parts.yaml"`)
   - Hash mismatch = npmDepsHash wrong (recompute with prefetch-npm-deps)
   - Evaluation error in module = syntax issue in module.nix

   Also try `nixos-rebuild build --flake .#acfs` for a full build test (this actually builds the entire system closure including Docker images — will take significant time but is the definitive test).

**Note on local path input:** Using `path:/data/projects/parts` means flake.lock will record an absolute path. Before deploying to the real server, this should be changed to `github:dangirsh/parts` (or whatever the remote URL is). This is acceptable for development — document this in the summary.
  </action>
