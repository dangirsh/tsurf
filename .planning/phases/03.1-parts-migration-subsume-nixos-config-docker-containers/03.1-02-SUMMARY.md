## Decisions Made
- Each workspace gets its own npmDepsHash (different lockfiles) rather than sharing root lockfile — no root package-lock.json exists
- postPatch copies workspace's package.json + package-lock.json to root for buildNpmPackage compatibility
- dontNpmBuild = true with custom buildPhase — tsc must run from workspace subdirectory for rootDir: ".." to work
- parts-tools includes coreutils in image contents for basic Unix utilities
- 100 maxLayers for parts-tools (complex image), default for parts-agent (simpler)

## Deviations from Plan

### Auto-fixed Issues

**1. npm EACCES on cache directory**
- **Found during:** Task 2 (parts-tools build, attempt 1)
- **Issue:** `npm error code EACCES` on `/nix/store/...-npm-deps/_cacache/tmp`
- **Fix:** Added `makeCacheWritable = true` to buildNpmPackage
- **Committed in:** `ad82e1a`

**2. signal-sdk network download blocked in Nix sandbox**
- **Found during:** Task 2 (parts-tools build, attempt 2)
- **Issue:** signal-sdk postinstall tries to download signal-cli from GitHub (`getaddrinfo EAI_AGAIN`)
- **Fix:** Added `npmFlags = [ "--ignore-scripts" ]` to skip all postinstall, then `npm rebuild better-sqlite3` in buildPhase to rebuild the native addon manually
- **Committed in:** `ad82e1a`

**3. Separate npmDepsHash per workspace**
- **Found during:** Task 1 (parts-agent build)
- **Issue:** Plan assumed shared root lockfile; workspaces have independent lockfiles
- **Fix:** Computed separate hashes via `prefetch-npm-deps` for each workspace lockfile
- **Committed in:** `ad82e1a`

**Total deviations:** 3 auto-fixed (2 blocking, 1 plan correction)
**Impact on plan:** All fixes necessary for Nix sandbox compatibility. No scope creep.

## Issues Encountered
- buildNpmPackage expects package-lock.json at src root — npm workspaces don't have one. Solved with postPatch copy pattern.
- signal-sdk's postinstall downloads binaries from GitHub — fundamentally incompatible with Nix sandbox. Solved by providing signal-cli via nixpkgs and skipping postinstall entirely.
- better-sqlite3 native addon needs manual rebuild after --ignore-scripts — identified by runtime error, fixed by adding `npm rebuild better-sqlite3` to buildPhase.
