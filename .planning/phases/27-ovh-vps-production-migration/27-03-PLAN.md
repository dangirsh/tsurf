**IMPORTANT: This task wipes the OVH VPS. Only run after human approval in Task 1.**

First, clear any existing SSH host key for the OVH IP (stock Ubuntu key will be invalid after deploy):
```bash
ssh-keygen -R 135.125.196.143
```

Run nixos-anywhere with password auth and pre-generated host key:
```bash
SSHPASS="6f2d2gQSdW2P" nix run github:nix-community/nixos-anywhere -- \
  --env-password \
  --extra-files tmp/ovh-host-keys \
  --flake .#ovh \
  root@135.125.196.143
```

**If kexec hangs** (no progress after 5+ minutes at kexec step), cancel and retry with older NixOS 22.11 kexec image:
```bash
SSHPASS="6f2d2gQSdW2P" nix run github:nix-community/nixos-anywhere -- \
  --env-password \
  --kexec "$(nix build --print-out-paths github:nix-community/nixos-images#packages.x86_64-linux.kexec-installer-nixos-22.11)/nixos-kexec-installer-noninteractive-x86_64-linux.tar.gz" \
  --extra-files tmp/ovh-host-keys \
  --flake .#ovh \
  root@135.125.196.143
```

After nixos-anywhere completes, wait 60 seconds for reboot, then verify SSH access:
```bash
ssh-keygen -R 135.125.196.143
ssh -o StrictHostKeyChecking=accept-new root@135.125.196.143 "hostname; uname -a; cat /etc/os-release"
```

If SSH connects and shows NixOS, the deploy succeeded. If SSH fails:
1. Check OVH VNC console for boot errors
2. If networking issue (DHCP vs static), may need OVH rescue mode to fix

**Post-deploy verification:**
```bash
ssh root@135.125.196.143 "systemctl is-system-running; systemctl --failed; ls /run/secrets/; tailscale status"
```

Check that:
- System is in "running" or "degraded" state (degraded is OK if Tailscale hasn't joined yet)
- No critical service failures
- `/run/secrets/` has decrypted secrets (sops-nix working)
- Tailscale status (may need manual `tailscale up` if auth key is stale)
