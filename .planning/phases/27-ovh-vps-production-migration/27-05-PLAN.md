<task type="auto">
  <name>Task 1: Migrate Docker state from Contabo to OVH</name>
  <files></files>
  <action>
**Step 1: Stop services on Contabo to get consistent state**

```bash
ssh root@neurosys "docker stop claw-swap-app claw-swap-caddy claw-swap-db parts-agent parts-tools"
```

**Step 2: Dump PostgreSQL for clean migration**

```bash
ssh root@neurosys "docker exec claw-swap-db pg_dumpall -U claw_swap > /tmp/claw-swap-dump.sql"
```

**Step 3: rsync Docker bind mount data from Contabo to OVH**

Transfer stateful data via Tailscale:
```bash
# claw-swap state (DB data, Caddy certs)
ssh root@neurosys "rsync -avz --progress /var/lib/claw-swap/ root@neurosys-prod:/var/lib/claw-swap/"

# parts state
ssh root@neurosys "rsync -avz --progress /var/lib/parts/ root@neurosys-prod:/var/lib/parts/"

# PostgreSQL dump as backup
ssh root@neurosys "rsync -avz /tmp/claw-swap-dump.sql root@neurosys-prod:/tmp/"
```

Note: If impermanence is active, paths may be under `/persist/var/lib/`. Adjust accordingly.

**Step 4: Restart services on OVH**

```bash
ssh root@neurosys-prod "docker restart claw-swap-db claw-swap-app claw-swap-caddy parts-agent parts-tools"
```

Wait for containers to stabilize:
```bash
ssh root@neurosys-prod "sleep 10 && docker ps --format 'table {{.Names}}\t{{.Status}}'"
```

**Step 5: Restart services on Contabo (staging)**

```bash
ssh root@neurosys "docker start claw-swap-db claw-swap-app claw-swap-caddy parts-agent parts-tools"
```

Contabo keeps running as staging — its data is now stale but that's fine for a staging environment.

**Step 6: Verify claw-swap is accessible on OVH**

```bash
curl -sk https://135.125.196.143/ -H "Host: claw-swap.com" | head -20
```

This should return the claw-swap page (Caddy handles TLS with the provided hostname).

- `curl -sk https://135.125.196.143/ -H "Host: claw-swap.com"` returns content
- PostgreSQL dump file exists on OVH at `/tmp/claw-swap-dump.sql`
  </verify>
  <done>
Docker state migrated from Contabo to OVH. claw-swap accessible on OVH IP. Both hosts have running containers.
  </done>
