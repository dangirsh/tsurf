**Phase A: Pre-deploy reconnaissance**

SSH into the OVH stock Ubuntu VPS to discover hardware details. Use `sshpass` for password auth:

```bash
# Install sshpass if not available
nix-shell -p sshpass --run 'sshpass -p "6f2d2gQSdW2P" ssh -o StrictHostKeyChecking=no root@135.125.196.143 "lsblk; echo ---; ip addr show; echo ---; ip route; echo ---; cat /etc/netplan/*.yaml 2>/dev/null || echo no-netplan; echo ---; ls /sys/firmware/efi 2>/dev/null && echo UEFI || echo BIOS; echo ---; cat /etc/os-release"'
```

Record in the summary:
- Disk device name (likely `/dev/sda` or `/dev/vda`)
- Primary network interface name (likely `ens3` or `eth0`)
- DHCP vs static IP config
- UEFI vs BIOS boot mode
- Prefix length and gateway from `ip route`

**Phase B: Generate SSH host key and derive age key**

```bash
mkdir -p tmp
ssh-keygen -t ed25519 -f tmp/ovh_ssh_host_ed25519_key -N "" -C "host_ovh"
```

Derive age public key:
```bash
nix-shell -p ssh-to-age --run 'ssh-to-age < tmp/ovh_ssh_host_ed25519_key.pub'
```

Record the age public key.

**Phase C: Update .sops.yaml with OVH host key**

Add `&host_ovh` anchor with the derived age key. Add a creation_rule for `secrets/ovh.yaml` encrypted with `admin + host_ovh`.

The updated `.sops.yaml` should look like:
```yaml
keys:
  - &admin          age1vma7w9nqlg9da8z60a99g8wv53ufakfmzxpkdnnzw39y34grug7qklz3xz
  - &host_neurosys  age1jgn7pqqf4hvalqdrzqysxtnsydd5urnuczrfm86umr7yfr8pu5gqqet2t3
  - &host_ovh       age1<derived-key>
creation_rules:
  - path_regex: secrets/neurosys\.yaml$
    key_groups:
      - age:
        - *admin
        - *host_neurosys
  - path_regex: secrets/ovh\.yaml$
    key_groups:
      - age:
        - *admin
        - *host_ovh
```

**Phase D: Create and encrypt secrets/ovh.yaml**

Create a plaintext `secrets/ovh.yaml` with the same secret keys as `secrets/neurosys.yaml`. Use the SAME values for secrets that are shared (API keys, B2 credentials), but generate a NEW Tailscale pre-auth key value (can be a placeholder — the user will replace it from the Tailscale admin console before deploy).

To get the current secret keys (not values — those are encrypted), inspect `secrets/neurosys.yaml` structure:
```bash
sops --decrypt secrets/neurosys.yaml 2>/dev/null | head -20
```

Write the plaintext YAML to `secrets/ovh.yaml` with the same structure and values (copy from decrypted neurosys.yaml, change tailscale-authkey to a placeholder). Then encrypt:
```bash
sops --encrypt --in-place secrets/ovh.yaml
```

**Phase E: Prepare --extra-files directory**

```bash
mkdir -p tmp/ovh-host-keys/persist/etc/ssh
cp tmp/ovh_ssh_host_ed25519_key tmp/ovh-host-keys/persist/etc/ssh/ssh_host_ed25519_key
chmod 600 tmp/ovh-host-keys/persist/etc/ssh/ssh_host_ed25519_key
```

The path uses `/persist/etc/ssh/` because both hosts use the impermanence module (Phase 21), which binds `/persist/etc/ssh` to `/etc/ssh`. OVH uses the same disko BTRFS config as Contabo, so `/persist` is definitive.
