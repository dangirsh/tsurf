  <action>
Add a new appendix to `docs/recovery-runbook.md` (after the last existing appendix) titled "Appendix: Tailnet Key Authority (TKA)" covering:

**Section 1: TKA Overview**
- What TKA does (self-custodied signing keys, coordination server compromise protection)
- Signing nodes: neurosys + one other device (minimum 2 required)
- TKA state stored in `/var/lib/tailscale` (persisted via impermanence)

**Section 2: TKA Initialization Procedure** (reference — the actual execution is Plan 23-02)
```bash
# Pre-checks
tailscale version   # Must be >= 1.90.8 (TS-2025-008 fix)
tailscale lock status  # Should show "disabled" before init
tailscale status   # Verify all nodes online

# Initialize (on a signing node)
tailscale lock init
# -> Generates 10 disablement secrets — SAVE THESE IMMEDIATELY

# Verify
tailscale lock status  # All nodes should show as signed
```

**Section 3: Signing New Devices**
```bash
# On an existing signing node:
tailscale lock sign nodekey:<key> tlpub:<key>
# Or pre-sign an auth key:
tailscale lock sign tskey-auth-XXXXCTRL-NNNNNN
```

**Section 4: Disablement Secret Storage**
- 10 secrets generated at init; only 1 needed to disable TKA
- Store in: (1) password manager, (2) offline backup (e.g., printed), (3) optionally enable Tailscale support backup during init
- Loss of ALL secrets + support backup disabled = permanent lock-in

**Section 5: Auth Key Rotation Policy**
- The `tailscale-authkey` in sops-nix is only used for initial setup or state loss (re-deploy from scratch)
- After TKA: auth keys must be pre-signed before use
- Rotation trigger: when the current key expires or if the key is suspected compromised
- Rotation procedure: generate new key in admin console -> `tailscale lock sign <key>` on a signing node -> update sops-nix secret -> deploy
- Recommendation: use tagged, one-time keys (not reusable)
- Node key expiry: 180 days default is reasonable for personal devices; servers with `tag:server` auto-disable expiry

**Section 6: TKA Disaster Recovery Implications**
- If `/var/lib/tailscale` is lost: node loses TKA signing state
- Recovery: re-authenticate via admin console, then re-sign with `tailscale lock sign` from another signing node
- If ALL signing nodes are lost: use a disablement secret to disable TKA, re-init

Keep the appendix concise with exact commands. Do not duplicate existing runbook content — reference earlier sections where appropriate (e.g., "See Phase 2: Restore State for `/var/lib/tailscale` restoration").
