TKA initialization MUST be performed interactively because:
1. It requires selecting signing nodes (at least 2 devices)
2. It generates 10 disablement secrets that must be immediately saved
3. The admin console may need to be used to initiate the lock (generating the `tailscale lock init` command with pre-populated keys)

**Steps for the user:**

1. **Go to Tailscale admin console** -> Settings -> Tailnet lock
2. **Enable Tailnet Lock** — select at least 2 signing nodes (neurosys + one laptop/desktop)
3. **Run the generated `tailscale lock init` command** on one of the signing nodes
4. **IMMEDIATELY save the 10 disablement secrets** to:
   - Password manager (primary)
   - Offline backup — printed or separate encrypted file (secondary)
   - Optionally enable Tailscale support backup (third)
5. **Verify on all nodes:** `tailscale lock status` — all should show as signed
6. **Verify on neurosys:** `ssh root@neurosys tailscale lock status`

**If any node was offline during init:**

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Harden ACL policy and configure key expiry</name>
  <action>
**ACL Policy Hardening (Admin Console -> Access Controls):**

Review the current ACL policy. For a personal tailnet, the recommended baseline using grants:

```json
{
  "tagOwners": {
    "tag:server": ["autogroup:admin"]
  },
  "grants": [
    {
      "src": ["autogroup:member"],
      "dst": ["tag:server"],
      "ip": ["*"]
    },
    {
      "src": ["autogroup:member"],
      "dst": ["autogroup:self"],
      "ip": ["*"]
    }
  ]
}
```

This means:
- Members can access servers tagged `tag:server` on all ports
- Members can access their own devices
- No other access is allowed (deny by default)

Apply `tag:server` to neurosys in the admin console (Machines -> neurosys -> Edit -> Tags).

**Key Expiry (Admin Console -> Settings -> Key expiry):**
- Personal devices (laptops, phones): keep 180-day default — forces periodic re-authentication
- Servers with `tag:server`: auto-disable expiry (tagged machines don't expire by default)
- Verify neurosys has `tag:server` and key expiry is disabled for it

**MagicDNS Review:**
- MagicDNS is secure by design — DNS resolution is local to each device
- No information leakage concerns for a personal tailnet
- Verify MagicDNS is enabled (Settings -> DNS -> MagicDNS)
- Verify `neurosys` resolves correctly: `tailscale status` should show the hostname

**Auth Key Review:**
- Check current `tailscale-authkey` properties in admin console
- If reusable: regenerate as one-time, tagged key
- Pre-sign the new key: `tailscale lock sign <new-auth-key>` (on a signing node)
- Update sops-nix: `sops secrets/neurosys.yaml` and replace the key value
- Note: this key is only used for fresh deployments (existing state persisted via impermanence)
