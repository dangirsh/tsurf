<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Harden ACL policy and configure key expiry</name>
  <action>
**ACL Policy Hardening (Admin Console -> Access Controls):**

Review the current ACL policy. For a personal tailnet, the recommended baseline using grants:

```json
{
  "tagOwners": {
    "tag:server": ["autogroup:admin"]
  },
  "grants": [
    {
      "src": ["autogroup:member"],
      "dst": ["tag:server"],
      "ip": ["*"]
    },
    {
      "src": ["autogroup:member"],
      "dst": ["autogroup:self"],
      "ip": ["*"]
    }
  ]
}
```

This means:
- Members can access servers tagged `tag:server` on all ports
- Members can access their own devices
- No other access is allowed (deny by default)

Apply `tag:server` to neurosys in the admin console (Machines -> neurosys -> Edit -> Tags).

**Key Expiry (Admin Console -> Settings -> Key expiry):**
- Personal devices (laptops, phones): keep 180-day default — forces periodic re-authentication
- Servers with `tag:server`: auto-disable expiry (tagged machines don't expire by default)
- Verify neurosys has `tag:server` and key expiry is disabled for it

**MagicDNS Review:**
- MagicDNS is secure by design — DNS resolution is local to each device
- No information leakage concerns for a personal tailnet
- Verify MagicDNS is enabled (Settings -> DNS -> MagicDNS)
- Verify `neurosys` resolves correctly: `tailscale status` should show the hostname

**Auth Key Pre-Signing (REQUIRED after TKA init — disaster recovery depends on this):**

TKA is now enabled (Task 2). Any auth key used for `nixos-anywhere` re-deploy must be pre-signed, or the new node will fail to join the tailnet. This step ensures disaster recovery works.

1. In the Tailscale admin console (Settings -> Keys), generate a new auth key:
   - **One-time** (not reusable)
   - **Tagged** with `tag:server`
   - **Expiry**: 90 days (re-generate before expiry or on next deploy)
2. Pre-sign the auth key from a signing node:
   ```bash
   tailscale lock sign tskey-auth-XXXXCTRL-NNNNNN
   ```
3. Update sops-nix with the new pre-signed auth key:
   ```bash
   sops secrets/neurosys.yaml
   # Replace the tailscale-authkey value with the new key
   ```
4. Deploy the updated secret:
   ```bash
   scripts/deploy.sh
   ```
5. Verify the key is accessible on the server:
   ```bash
   ssh root@neurosys cat /run/secrets/tailscale-authkey | head -c 20
   # Should show the first 20 chars of the new key
   ```

Note: This key is only used for fresh deployments (existing tailscale state is persisted). But without a pre-signed key, disaster recovery via `nixos-anywhere` would fail to join the tailnet.
