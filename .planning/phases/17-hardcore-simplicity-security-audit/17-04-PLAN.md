**Part C -- Harden audit log against tamper.**

In `modules/agent-compute.nix`, change the systemd.tmpfiles rule for the audit log directory from:
```nix
"d /data/projects/.agent-audit 0750 dangirsh users -"
```
to:
```nix
"d /data/projects/.agent-audit 0750 root root -"
```

And modify the agent-spawn script's audit logging section. Instead of directly writing to spawn.log (which requires dangirsh write access), use a helper approach:

Option 1 (simplest): Change the tmpfiles rule to make the directory owned by root but with a group-writable sticky bit. Actually, the simplest effective mitigation: make the log file append-only via `chattr +a`. But this requires root and NixOS activation.

Option 2 (practical): Add a tmpfiles rule that creates the log file as root-owned, mode 0662 (group-writable), and use `tee -a` with a root-owned file. But agents run as dangirsh...

**Go with the simplest practical approach:** Keep the directory owned by dangirsh (needed for agent-spawn to write), but add a comment documenting the tamper risk and noting that for production audit integrity, the spawn.log should be forwarded to a root-owned journald entry or syslog. For now, the existing approach is acceptable because:
- The audit log's primary purpose is operational visibility, not forensic integrity
- A compromised agent would need to know about and target the audit log specifically
- The sandbox itself prevents most attack vectors

Update the comment in agent-compute.nix above the tmpfiles rule:
```nix
# Pre-create audit log directory for agent-spawn.
# NOTE (SEC-17-04): spawn.log is writable by dangirsh -- a compromised agent
# could tamper with it. For forensic-grade audit integrity, forward spawn events
# to journald (systemd-cat) in a future hardening pass. Current risk: LOW
# (operational log, not security boundary).
```

Also update the `printf` line in agent-spawn to additionally log to journald for tamper-resistant audit trail:
```bash
printf "%s SPAWN agent=%s name=%s project=%s sandbox=%s\n" \
  "$(date -Iseconds)" "$AGENT" "$NAME" "$PROJECT_DIR" "$SANDBOX_STATE" >> "$AUDIT_DIR/spawn.log"
# Tamper-resistant copy via journald (root-owned journal, agents cannot modify)
printf "AGENT_SPAWN agent=%s name=%s project=%s sandbox=%s" \
  "$AGENT" "$NAME" "$PROJECT_DIR" "$SANDBOX_STATE" | systemd-cat -t agent-spawn -p info
```

This gives us dual logging: the existing spawn.log for easy reading, plus journald for tamper-resistant audit.
