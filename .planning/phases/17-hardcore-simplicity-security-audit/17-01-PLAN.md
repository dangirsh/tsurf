**SEC2 — Kernel sysctl hardening:** Add the following to `modules/base.nix`:

```nix
# @decision SEC-17-01: Standard Linux server kernel hardening via sysctl
boot.kernel.sysctl = {
  "kernel.dmesg_restrict" = 1;           # Restrict dmesg to root
  "kernel.kptr_restrict" = 2;            # Hide kernel pointers from non-root
  "kernel.unprivileged_bpf_disabled" = 1; # Disable unprivileged eBPF
  "net.core.bpf_jit_harden" = 2;        # Harden eBPF JIT compiler
  "net.ipv4.conf.all.accept_redirects" = false;    # Prevent ICMP redirect MITM
  "net.ipv4.conf.default.accept_redirects" = false;
  "net.ipv4.conf.all.send_redirects" = false;
  "net.ipv6.conf.all.accept_redirects" = false;
  "net.ipv6.conf.default.accept_redirects" = false;
  "net.ipv4.conf.all.log_martians" = true;  # Log suspicious packets
};
```

Place this block after the `nix.gc` block and before `environment.systemPackages`. These are standard Linux server hardening settings from Ryan Seipp's NixOS hardening guide and Lynis recommendations. They restrict kernel information leakage and harden network stack against ICMP-based attacks.

**SEC10 — llm-agents supply chain fix:** In `flake.nix`, change the `llm-agents` input from:
```nix
llm-agents = {
  url = "github:numtide/llm-agents.nix";
};
```
to:
```nix
llm-agents = {
  url = "github:numtide/llm-agents.nix";
  inputs.nixpkgs.follows = "nixpkgs";
};
```

This forces llm-agents to use the same nixpkgs (nixos-25.11) as the rest of the system, eliminating a separate nixpkgs-unstable evaluation (supply chain concern, closure size, evaluation time).

**IMPORTANT:** After adding the follows, run `nix flake lock --update-input llm-agents` to recompute the lock, then `nix flake check`. If `nix flake check` fails due to llm-agents packages being incompatible with nixos-25.11 (Pitfall 3 from research), REVERT the follows change and accept the separate nixpkgs. Document the outcome either way.
