    Create docs/private-overlay.md with the following content and structure:

    # Private Overlay Pattern

    A one-paragraph intro: this public repo provides generic NixOS infrastructure modules. Personal config (secrets, private services, host IPs, your username) goes in a private flake repo that imports this one. The public repo exports nixosModules.default for that purpose.

    ## Structure

    A file tree showing the private repo layout:
    ```
    neurosys-private/
      flake.nix              # imports neurosys public + wires personal config
      secrets/
        neurosys.yaml        # sops-encrypted secrets
        ovh.yaml
      .sops.yaml             # age key recipients
      modules/
        private-default.nix  # import hub for private modules
        home-assistant.nix   # personal home automation
        repos.nix            # private repos to clone on activation
        spacebot.nix         # personal services
        # ... more personal modules
      hosts/
        neurosys/
          default.nix        # extends public host config with private config
    ```

    ## flake.nix skeleton

    A complete, annotated flake.nix that:
    - Takes `neurosys` as a flake input pointing to the public repo URL (github:your-github/neurosys)
    - Takes all the same standard inputs (nixpkgs, home-manager, sops-nix, disko, etc.) with follows
    - Also takes private-service inputs (personal-agent-runtime, claw-swap, your-site, etc.)
    - Defines commonModules as: `neurosys.nixosModules.default` plus private nixosModules from private inputs, plus the private-default.nix module hub
    - Defines mkHost the same way as the public repo
    - Defines nixosConfigurations.neurosys = mkHost ./hosts/neurosys with the private host config

    The skeleton flake.nix must be syntactically correct Nix (valid enough to read, with TODO comments where users fill in their details).

    ## modules/private-default.nix example

    Show a simple imports list referencing the private modules.

    ## hosts/neurosys/default.nix example

    Show how the private host config extends the public one:
    - imports = [ neurosys.nixosModules.default ]; is NOT needed (already in commonModules)
    - Sets networking.hostName, static IP block, sops.defaultSopsFile = ../../secrets/neurosys.yaml
    - Sets home-manager.users.myactualuser = import ./home
    - Shows where to add host-specific srvos overrides

    ## Secrets

    Brief section: copy the .sops.yaml pattern from the public repo docs, generate your age keys (ssh-to-age from host key), add your admin age key, run sops to create secrets files. Reference the sops-nix README for full setup.

    ## Adding a private service module example

    Show a minimal home-assistant.nix that follows the same pattern as the public module (openFirewall = false, Tailscale-only), so users understand the convention.

    ## Wiring the secret proxy for private projects

    Show how to extend agent-compute.nix behavior in the private overlay by adding a NixOS module that overrides the agent-spawn script or adds a wrapper — or note that the user should edit agent-compute.nix in the public fork for project-specific routing.

    The document should be dense and technical, matching the stereos README style: no hand-holding, tables where possible, concrete code blocks. Target: 100-150 lines.
