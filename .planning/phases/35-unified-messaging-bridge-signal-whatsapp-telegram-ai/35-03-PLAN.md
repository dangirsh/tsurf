# Plan 35-03: AI Read Bot + Historical Ingest Pipeline

## Goal

Create the `dangirsh/matrix-bridge-tools` repository containing a matrix-nio
Python bot that reads all bridged Matrix rooms and forwards messages to Spacebot
for AI indexing. Also create one-time ingest scripts for historical message data
from Telegram (JSON export), Signal (Android backup), and WhatsApp (text export).
Wire the bot into neurosys as a systemd service via the repos.nix clone pattern.

## Decisions Made (from Research + User Decision)

- **New repo:** `dangirsh/matrix-bridge-tools` on GitHub (USER DECISION: "consider
  a new repo for this that neurosys pulls in"). Created as a standard Python
  project with pyproject.toml.
- **Integration pattern:** repos.nix clone to `/data/projects/matrix-bridge-tools`
  (matches home-assistant-config precedent). Bot runs as systemd service pointing
  to the cloned repo's Python script.
- **Bot framework:** matrix-nio (`python3Packages.matrix-nio` in nixpkgs).
  Simplest API for read-only message sync. ~100-150 lines for the bot.
- **Bot user:** `@aibot:neurosys.local` registered on Conduit (one-time manual
  step, same pattern as admin user registration). Access token stored in sops
  as `matrix-bot-token`.
- **Ingest target:** Spacebot LanceDB. Open question: does Spacebot expose an
  HTTP ingest endpoint, or must we write directly to `/var/lib/spacebot/`?
  First task includes investigating the Spacebot data model.
- **Historical ingest:** Three one-time scripts run manually after deploying the
  bot. They process exported data and write to Spacebot. Not systemd services.
- **Python runtime:** `pkgs.python3.withPackages` providing matrix-nio for the
  bot. Ingest scripts may need additional packages (sqlite3 is stdlib, lancedb
  may need pip install or a Nix overlay).

## must_haves

1. `dangirsh/matrix-bridge-tools` GitHub repo exists with bot/reader.py and ingest/ scripts
2. `modules/repos.nix` includes `dangirsh/matrix-bridge-tools` in the clone list
3. `modules/matrix.nix` has `systemd.services.matrix-reader-bot` service definition
4. `modules/secrets.nix` declares `sops.secrets."matrix-bot-token"` with explicit `sopsFile`
5. `secrets/neurosys.yaml` contains `matrix-bot-token` placeholder
6. `nix flake check` passes
7. After deploy: `systemctl status matrix-reader-bot` shows the service
8. After bot user creation + token: bot syncs messages from bridged rooms
9. Ingest scripts exist and have been tested with sample data

          # Implementation depends on 35-03-A findings:

          # Option C: Append to staging JSON file

      **1. ingest/telegram.py — Telegram JSON export parser**

      Input: Telegram Desktop JSON export directory (containing `result.json`)

      ```python
      #!/usr/bin/env python3
      """Ingest Telegram JSON export into Spacebot."""

      import json
      import sys
      from pathlib import Path
      from datetime import datetime


      def parse_text(text_field):
          """Flatten Telegram text field (can be str or list of entities)."""
          if isinstance(text_field, str):
              return text_field
          if isinstance(text_field, list):
              return "".join(
                  part if isinstance(part, str) else part.get("text", "")
                  for part in text_field
              )
          return ""


      def ingest_telegram(export_dir: str):
          """Parse Telegram JSON export and yield message dicts."""
          result_path = Path(export_dir) / "result.json"
          with open(result_path) as f:
              data = json.load(f)

          for chat in data.get("chats", {}).get("list", []):
              chat_name = chat.get("name", "Unknown")
              chat_type = chat.get("type", "unknown")

              for msg in chat.get("messages", []):
                  if msg.get("type") != "message":
                      continue
                  text = parse_text(msg.get("text", ""))
                  if not text.strip():
                      continue

                  yield {
                      "source": "telegram",
                      "chat_name": chat_name,
                      "chat_type": chat_type,
                      "sender": msg.get("from", "Unknown"),
                      "text": text,
                      "timestamp": msg.get("date", ""),
                      "message_id": msg.get("id"),
                  }


      if __name__ == "__main__":
          if len(sys.argv) != 2:
              print(f"Usage: {sys.argv[0]} <telegram-export-dir>")
              sys.exit(1)

          count = 0
          for msg in ingest_telegram(sys.argv[1]):
              # Forward to Spacebot (implementation depends on 35-03-A)
              count += 1

          print(f"Ingested {count} messages from Telegram export")
      ```

      **2. ingest/signal.py — Signal backup parser**

      Input: Decrypted Signal backup directory (from `signalbackup-tools`)

      Pre-requisite: User runs `signalbackup-tools` to decrypt the .backup file:
      ```bash
      signalbackup-tools signal.backup <30-digit-passphrase> --output /data/projects/matrix-bridge-tools/tmp/signal-export/
      ```

      The script reads the `database.sqlite` file from the export, joins the
      `sms`/`mms` tables with `recipient` to get contact names, and yields
      message dicts.

      **Key SQLite queries:**
      ```sql
      -- Get conversations with contact names
      SELECT t.*, r.system_display_name as contact_name
      FROM thread t
      JOIN recipient r ON t.recipient_id = r._id;

      -- Get SMS messages
      SELECT s.body, s.date, s.type, r.system_display_name as sender
      FROM sms s
      JOIN recipient r ON s.recipient_id = r._id
      WHERE s.body IS NOT NULL AND s.body != '';

      -- Get MMS messages (may have text body)
      SELECT m.body, m.date, m.msg_box, r.system_display_name as sender
      FROM mms m
      JOIN recipient r ON m.recipient_id = r._id
      WHERE m.body IS NOT NULL AND m.body != '';
      ```

      **3. ingest/whatsapp.py — WhatsApp .txt export parser**

      Input: WhatsApp exported .txt file (from WhatsApp > Chat > Export)

      ```python
      #!/usr/bin/env python3
      """Ingest WhatsApp .txt chat export into Spacebot."""

      import re
      import sys
      from pathlib import Path


      # Regex patterns for different WhatsApp export formats
      # MM/DD/YYYY, H:MM AM/PM - Sender: Message
      PATTERN_US = re.compile(
          r"^(\d{1,2}/\d{1,2}/\d{2,4}),?\s+(\d{1,2}:\d{2}(?::\d{2})?\s*(?:AM|PM)?)\s*-\s*([^:]+):\s*(.*)"
      )
      # DD/MM/YYYY, HH:MM - Sender: Message (24h format)
      PATTERN_EU = re.compile(
          r"^(\d{1,2}/\d{1,2}/\d{2,4}),?\s+(\d{1,2}:\d{2}(?::\d{2})?)\s*-\s*([^:]+):\s*(.*)"
      )


      def ingest_whatsapp(txt_path: str):
          """Parse WhatsApp .txt export and yield message dicts."""
          current_msg = None
          chat_name = Path(txt_path).stem  # Use filename as chat name

          with open(txt_path, encoding="utf-8") as f:
              for line in f:
                  line = line.rstrip()
                  match = PATTERN_US.match(line) or PATTERN_EU.match(line)

                  if match:
                      if current_msg:
                          yield current_msg
                      date, time, sender, text = match.groups()
                      if text == "<Media omitted>":
                          continue
                      current_msg = {
                          "source": "whatsapp",
                          "chat_name": chat_name,
                          "sender": sender.strip(),
                          "text": text,
                          "timestamp": f"{date} {time}",
                          "message_id": None,
                      }
                  elif current_msg:
                      # Continuation line (multi-line message)
                      current_msg["text"] += "\n" + line

          if current_msg:
              yield current_msg


      if __name__ == "__main__":
          if len(sys.argv) != 2:
              print(f"Usage: {sys.argv[0]} <whatsapp-export.txt>")
              sys.exit(1)

          count = 0
          for msg in ingest_whatsapp(sys.argv[1]):
              count += 1

          print(f"Ingested {count} messages from WhatsApp export")
      ```

      on task 35-03-A findings).

## Risks and Mitigations

### Spacebot ingest API uncertainty (MEDIUM)

The biggest unknown is how to get data into Spacebot's LanceDB. If there is no
HTTP ingest endpoint, direct LanceDB writes require understanding the embedding
schema (vector dimensions, metadata fields). Mitigation: task 35-03-A
investigates this first. Worst case: write messages to a simple JSON staging
file and defer Spacebot integration to a future phase.

### matrix-nio package availability (LOW)

`python3Packages.matrix-nio` is confirmed in nixpkgs-25.11 but may have version
compatibility issues with Conduit's Matrix API version. Mitigation: matrix-nio
supports CS API v1.1+ which Conduit implements. If issues arise, pin a specific
version or use a direct HTTP client as fallback.

### repos.nix clone timing (LOW)

The repos.nix activation script clones repos once. If the matrix-bridge-tools
repo is created in task 35-03-B and the deploy happens before the repo exists
on GitHub, the clone will fail (with a warning) and succeed on the next
activation. This is the expected behavior -- clone-only, never pull.

### Signal backup format variations (LOW)

The SQLite schema in Signal Android backups varies between Signal versions.
The ingest script must handle missing columns gracefully. Mitigation: test
with the user's actual Signal backup before deploying.

## Notes for Implementer

- The matrix-bridge-tools repo is created OUTSIDE the neurosys worktree. It is
  a separate GitHub repository. Create it with `gh repo create` as shown in
  task 35-03-B.
- The Python scripts in the new repo do NOT need Nix packaging. They run via
  `pkgs.python3.withPackages` which provides the runtime environment.
- The ingest scripts are one-time tools, not persistent services. They do not
  need systemd units. Run them manually via SSH to the server.
- The `signalbackup-tools` package is in nixpkgs. The user can access it via
  `nix-shell -p signalbackup-tools` on the server.
- If the Spacebot ingest investigation (35-03-A) reveals no clean integration
  path, consider a simpler approach: the bot writes messages to a SQLite
  database at `/var/lib/matrix-reader-bot/messages.db`. This provides immediate
  value (all DMs in one queryable store) even without Spacebot integration.
  Spacebot integration can be added later when its ingest API is better understood.
