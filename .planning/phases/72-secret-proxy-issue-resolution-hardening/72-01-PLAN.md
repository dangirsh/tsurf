    **3. Graceful shutdown (DEG-04):**
    Replace:
    ```rust
    axum::serve(listener, app).await.unwrap();
    ```
    With:
    ```rust
    axum::serve(listener, app)
        .with_graceful_shutdown(shutdown_with_timeout(config.shutdown_timeout_secs))
        .await
        .unwrap();
    ```

    This bounds the drain period: after receiving SIGTERM/SIGINT, the proxy gives in-flight connections up to `shutdown_timeout_secs` (default 30s) to complete, then exits. This prevents systemd's `TimeoutStopSec` from racing against an unbounded drain.

    Add a private async helper alongside `shutdown_signal`:
    ```rust
    async fn shutdown_with_timeout(secs: u64) {
        let _ = tokio::time::timeout(
            std::time::Duration::from_secs(secs),
            shutdown_signal(),
        ).await;
    }
    ```

    Add a private async function at the bottom of main.rs (outside `main`):
    ```rust
    async fn shutdown_signal() {
        use tokio::signal;
        let ctrl_c = async {
            signal::ctrl_c().await.expect("failed to install CTRL+C handler");
        };

        #[cfg(unix)]
        let terminate = async {
            signal::unix::signal(signal::unix::SignalKind::terminate())
                .expect("failed to install SIGTERM handler")
                .recv()
                .await;
        };

        #[cfg(not(unix))]
        let terminate = std::future::pending::<()>();

        tokio::select! {
            _ = ctrl_c => {},
            _ = terminate => {},
        }
        tracing::info!("shutdown signal received, draining connections");
    }
    ```

    No new dependencies needed — `tokio::signal` is in scope via `tokio = { features = ["full"] }`. The `std::time::Duration` is in std. Imports at top of file: add `use std::time::Duration;` if convenient, but `std::time::Duration::from_secs` inline is also fine and avoids changing the use block.
  </action>
