    ```markdown
    ## Networking

    The proxy supports two Docker networking modes:

    ### Bridge network (recommended)

    Set `bind = "0.0.0.0"` in `config.toml` so the proxy listens on all interfaces within the container:

    ```toml
    port = 9091
    bind = "0.0.0.0"
    placeholder = "sk-ant-api03-placeholder"

    [[secret]]
    header = "x-api-key"
    name = "anthropic"
    file = "/run/secrets/anthropic-api-key"
    allowed_domains = ["api.anthropic.com"]
    ```

    Sibling containers can then reach the proxy at `http://secret-proxy:9091` (using the service name as DNS):

    ```yaml
    services:
      secret-proxy:
        image: ghcr.io/dangirsh/nix-secret-proxy:latest
        volumes:
          - ./config.toml:/etc/secret-proxy/config.toml:ro
          - /run/secrets/anthropic-api-key:/run/secrets/anthropic-api-key:ro
        command: ["--config", "/etc/secret-proxy/config.toml"]
        restart: unless-stopped

      my-agent:
        image: my-agent:latest
        environment:
          ANTHROPIC_BASE_URL: "http://secret-proxy:9091"
          ANTHROPIC_API_KEY: "sk-ant-api03-placeholder"
        depends_on:
          - secret-proxy
    ```

    Docker's default bridge network isolates containers from the host network; binding `0.0.0.0` inside the container does not expose the proxy to the host machine's external interfaces.

    ### Host network (alternative)

    If you cannot change `config.toml` (e.g., using a pre-built image with a fixed config), use `network_mode: host` for both containers. This shares the host network namespace so `127.0.0.1` is reachable from sibling containers:
    ```
