# Plan 32-02: NixOS Service Module + Secrets + State Pre-Seeding

## Goal

Create `modules/automaton.nix` as a systemd service that runs the Conway
Automaton agent on neurosys. Wire sops-nix secrets for the Conway API key.
Pre-seed the state directory (`/var/lib/automaton/.automaton/`) with
configuration files so the interactive wizard is bypassed. Verify the agent
starts and completes at least one reasoning turn.

## Decisions Made (from Research + User Decisions)

- **Host**: neurosys (Contabo, 18 vCPU / 96 GB RAM) -- not OVH
- **User**: Dedicated `automaton` system user, home `/var/lib/automaton`
- **State directory**: `HOME=/var/lib/automaton` so `~/.automaton/` resolves
  to `/var/lib/automaton/.automaton/`
- **Inference**: BYOK via existing anthropic-secret-proxy on port 9091
  (`ANTHROPIC_BASE_URL=http://127.0.0.1:9091` + placeholder API key)
- **Conway API key**: Stored in sops-nix, injected into `automaton.json`
  config via activation script. Enables Conway Cloud tools (domains, x402,
  credits) without running the agent in a Conway sandbox.
- **Genesis prompt**: Placeholder text -- user fills in manually before
  first run. The service will NOT start automatically until the user
  confirms the genesis prompt is set (enabled via a manual `systemctl start`
  or by setting `wantedBy` after configuration).
- **Wallet**: Generated fresh on first activation if not pre-existing.
  The activation script creates `wallet.json` with a new random key if the
  file does not exist. Once created, it persists across rebuilds.
- **Survival tier**: With a Conway API key provisioned, the agent gets
  authentic survival pressure from credits. No patching of `getSurvivalTier`.
- **No ports exposed**: The automaton is a headless background process making
  only outbound connections. No changes to `networking.nix`.
- **Module location**: Own file `modules/automaton.nix` (>20 lines, justified)

## must_haves

1. `modules/automaton.nix` exists with systemd service `conway-automaton`
2. `modules/default.nix` imports `./automaton.nix`
3. `modules/secrets.nix` declares `sops.secrets."conway-api-key"` with `owner = "automaton"`
4. `sops.templates."automaton-env"` renders `ANTHROPIC_BASE_URL` and placeholder `ANTHROPIC_API_KEY`
5. System user `automaton` exists with `isSystemUser = true`, group `automaton`, home `/var/lib/automaton`
6. Activation script pre-seeds `/var/lib/automaton/.automaton/automaton.json` with correct config
7. Activation script copies `constitution.md` from the automaton package
8. Service depends on `network-online.target` and `anthropic-secret-proxy.service`
9. `HOME=/var/lib/automaton` is set in the service environment
10. `nix flake check` passes
11. After deployment: `systemctl status conway-automaton` shows the service (may be inactive until user enables)
12. After manual start: `sqlite3 /var/lib/automaton/.automaton/state.db "SELECT count(*) FROM turns"` returns >= 1

    <description>
      Create `/data/projects/neurosys/modules/automaton.nix`.

      This is the main module file. Structure:

      ```nix
      # modules/automaton.nix
      # Conway Automaton — self-hosted autonomous AI agent runtime
      #
      # @decision AUTO-03: Agent runs on neurosys hardware, not Conway Cloud sandbox.
      # @rationale: Eliminates Conway Cloud sandbox fees. Agent uses Conway API key
      #   for Cloud tools (domains, x402, credits) but execution is local. BYOK
      #   Anthropic inference via secret proxy on port 9091.
      #
      # @decision AUTO-04: HOME=/var/lib/automaton for state isolation.
      # @rationale: The automaton hardcodes ~/.automaton/ for wallet and config.
      #   Setting HOME to the StateDirectory keeps all state under /var/lib/automaton.
      #
      # @decision AUTO-05: Genesis prompt is a placeholder; user fills in before first run.
      # @rationale: The genesis prompt defines the agent's mission and personality.
      #   This is a creative decision that should not be automated.
      #
      # @decision AUTO-06: Wallet generated on first activation, persists across rebuilds.
      # @rationale: The EVM wallet private key is generated once and never overwritten.
      #   Storing in /var/lib/automaton/.automaton/wallet.json (persisted by impermanence
      #   or StateDirectory) ensures the agent keeps its identity across NixOS rebuilds.

      { config, pkgs, inputs, lib, ... }:

      let
        automaton-pkg = pkgs.callPackage ../packages/automaton.nix {
          src = inputs.automaton;
        };

        automatonDir = "/var/lib/automaton/.automaton";

        # Default automaton.json configuration
        # The genesis prompt is a placeholder — user MUST edit before first run
        automatonConfig = builtins.toJSON {
          name = "neurosys-agent";
          genesisPrompt = "PLACEHOLDER — edit /var/lib/automaton/.automaton/automaton.json before starting the service";
          creatorAddress = "0x0000000000000000000000000000000000000000";
          registeredWithConway = false;
          sandboxId = "";
          conwayApiUrl = "https://api.conway.tech";
          conwayApiKey = "";
          anthropicApiKey = "placeholder-for-secret-proxy";
          inferenceModel = "claude-sonnet-4-6";
          maxTokensPerTurn = 4096;
          heartbeatConfigPath = "${automatonDir}/heartbeat.yml";
          dbPath = "${automatonDir}/state.db";
          logLevel = "info";
          walletAddress = "";
          version = "0.2.0";
          skillsDir = "${automatonDir}/skills";
          maxChildren = 0;
          treasuryPolicy = {
            maxSingleTransferCents = 5000;
            maxHourlyTransferCents = 10000;
            maxDailyTransferCents = 50000;
            minimumReserveCents = 5000;
            maxX402PaymentCents = 500;
            x402AllowedDomains = [ "*" ];
            transferCooldownMs = 60000;
            maxTransfersPerTurn = 3;
            maxInferenceDailyCents = 5000;
            requireConfirmationAboveCents = 1000;
          };
        };

        # Default heartbeat.yml configuration
        heartbeatYml = ''
          # Conway Automaton heartbeat schedule
          # Intervals in seconds
          tasks:
            heartbeat_ping:
              interval: 300
              enabled: true
            check_credits:
              interval: 3600
              enabled: true
            check_usdc_balance:
              interval: 3600
              enabled: true
            check_for_updates:
              interval: 86400
              enabled: false
            health_check:
              interval: 300
              enabled: true
            soul_reflection:
              interval: 86400
              enabled: true
            report_metrics:
              interval: 3600
              enabled: true
        '';

        # Default SOUL.md — minimal identity document
        soulMd = ''
          # Agent Identity

          I am a sovereign AI agent running on neurosys infrastructure.
          My mission is defined by my genesis prompt.
          I operate autonomously with a funded Ethereum wallet.

          ## Capabilities
          - Full internet access for research and API calls
          - Conway Cloud tools (domains, x402 payments, credits)
          - BYOK Anthropic inference via secret proxy
          - SQLite state persistence
          - Git-based state versioning
          - Self-modification of skills and heartbeat schedule

          ## Values
          - Survive: earn more than I spend
          - Create value: build things others will pay for
          - Evolve: improve my own capabilities over time
        '';
      in {

        # --- System user ---
        users.users.automaton = {
          isSystemUser = true;
          group = "automaton";
          home = "/var/lib/automaton";
          createHome = true;
        };
        users.groups.automaton = {};

        # --- Sops template for environment ---
        sops.templates."automaton-env" = {
          owner = "automaton";
          content = ''
            ANTHROPIC_BASE_URL=http://127.0.0.1:9091
            ANTHROPIC_API_KEY=placeholder-for-secret-proxy
          '';
        };

        # --- Activation script: pre-seed state directory ---
        system.activationScripts.automaton-state = {
          text = ''
            # Create state directory structure
            mkdir -p ${automatonDir}/skills
            chown -R automaton:automaton /var/lib/automaton

            # Write automaton.json only if it does not exist (preserve user edits)
            if [ ! -f ${automatonDir}/automaton.json ]; then
              echo '${automatonConfig}' > ${automatonDir}/automaton.json
              chown automaton:automaton ${automatonDir}/automaton.json
              chmod 0600 ${automatonDir}/automaton.json
            fi

            # Inject Conway API key from sops into automaton.json
            # The config file stores the key directly (not env var)
            if [ -f ${config.sops.secrets."conway-api-key".path} ]; then
              CONWAY_KEY=$(cat ${config.sops.secrets."conway-api-key".path})
              if [ -n "$CONWAY_KEY" ] && [ "$CONWAY_KEY" != "cnwy_k_PLACEHOLDER" ]; then
                ${pkgs.jq}/bin/jq --arg key "$CONWAY_KEY" '.conwayApiKey = $key' \
                  ${automatonDir}/automaton.json > ${automatonDir}/automaton.json.tmp \
                  && mv ${automatonDir}/automaton.json.tmp ${automatonDir}/automaton.json
                chown automaton:automaton ${automatonDir}/automaton.json
                chmod 0600 ${automatonDir}/automaton.json
              fi
            fi

            # Generate wallet.json if it does not exist
            # Creates a random EVM private key (32 bytes hex)
            if [ ! -f ${automatonDir}/wallet.json ]; then
              PRIVKEY=$(${pkgs.openssl}/bin/openssl rand -hex 32)
              echo "{\"privateKey\": \"0x$PRIVKEY\"}" > ${automatonDir}/wallet.json
              chown automaton:automaton ${automatonDir}/wallet.json
              chmod 0600 ${automatonDir}/wallet.json
            fi

            # Write heartbeat.yml only if it does not exist
            if [ ! -f ${automatonDir}/heartbeat.yml ]; then
              cat > ${automatonDir}/heartbeat.yml << 'HEARTBEAT_EOF'
            ${heartbeatYml}
            HEARTBEAT_EOF
              chown automaton:automaton ${automatonDir}/heartbeat.yml
            fi

            # Write SOUL.md only if it does not exist (agent may self-modify)
            if [ ! -f ${automatonDir}/SOUL.md ]; then
              cat > ${automatonDir}/SOUL.md << 'SOUL_EOF'
            ${soulMd}
            SOUL_EOF
              chown automaton:automaton ${automatonDir}/SOUL.md
            fi

            # Always update constitution.md from package (immutable rules)
            cp ${automaton-pkg}/lib/node_modules/@conway/automaton/constitution.md \
              ${automatonDir}/constitution.md
            chown automaton:automaton ${automatonDir}/constitution.md
            chmod 0444 ${automatonDir}/constitution.md

            # Initialize git repo for state versioning if not present
            if [ ! -d ${automatonDir}/.git ]; then
              ${pkgs.git}/bin/git -C ${automatonDir} init
              ${pkgs.git}/bin/git -C ${automatonDir} config user.email "automaton@neurosys"
              ${pkgs.git}/bin/git -C ${automatonDir} config user.name "Conway Automaton"
              # Create .gitignore to exclude secrets and DB
              cat > ${automatonDir}/.gitignore << 'GITIGNORE_EOF'
            wallet.json
            config.json
            state.db
            state.db-wal
            state.db-shm
            *.log
            GITIGNORE_EOF
              chown -R automaton:automaton ${automatonDir}/.git ${automatonDir}/.gitignore
            fi
          '';
          deps = [ "setupSecrets" ];
        };

        # --- Systemd service ---
        systemd.services.conway-automaton = {
          description = "Conway Automaton autonomous agent runtime";
          after = [
            "network-online.target"
            "anthropic-secret-proxy.service"
            "sops-nix.service"
          ];
          wants = [ "network-online.target" ];
          # NOT in wantedBy — user must manually start after setting genesis prompt
          # Uncomment the next line after configuring the genesis prompt:
          # wantedBy = [ "multi-user.target" ];

          serviceConfig = {
            ExecStart = "${automaton-pkg}/bin/automaton --run";
            EnvironmentFile = config.sops.templates."automaton-env".path;
            User = "automaton";
            Group = "automaton";
            StateDirectory = "automaton";
            WorkingDirectory = "/var/lib/automaton";
            Environment = [
              "HOME=/var/lib/automaton"
              "NODE_ENV=production"
            ];
            Restart = "on-failure";
            RestartSec = "30s";

            # Systemd hardening
            NoNewPrivileges = true;
            ProtectSystem = "strict";
            ReadWritePaths = [ "/var/lib/automaton" ];
          };
        };
      }
      ```

      **Key design decisions in the activation script:**
      - All config files use "create-if-absent" pattern: the activation script
        never overwrites existing files (except `constitution.md` which is
        immutable). This preserves user edits and agent self-modifications.
      - The Conway API key is injected via `jq` from the sops secret into
        `automaton.json` on every activation. This keeps the key in sync
        with sops without requiring the user to manually edit JSON.
      - The wallet is generated with `openssl rand -hex 32` as a fallback.
        The automaton framework normally generates this via its own code, but
        pre-generating it avoids the interactive wizard dependency.
      - The service is NOT in `wantedBy` by default. The user must either
        uncomment `wantedBy` or run `systemctl start conway-automaton` after
        configuring the genesis prompt. This prevents the agent from starting
        with a placeholder prompt.

      Verify: The file evaluates without syntax errors (checked via
      `nix flake check` in task 32-02-D).

## Notes for Implementer

- The `automaton-pkg` reference uses `inputs.automaton` which comes from
  `specialArgs = { inherit inputs; }` in the flake. This is the same pattern
  used by the claw-swap module with `self + "/nix/..."`.
- The activation script runs as root (standard for NixOS activation scripts).
  It creates files owned by the `automaton` user via explicit `chown`.
- The `jq` injection of the Conway API key on every activation is intentional:
  it ensures the key stays in sync with sops without manual JSON editing.
  The rest of `automaton.json` is NOT overwritten (only `conwayApiKey` field).
- The wallet generation uses `openssl rand` rather than the automaton's own
  wallet generation code because the activation script runs before the service.
  The generated key format (`{"privateKey": "0x..."}`) matches what the
  automaton framework expects.
- If OVH evaluation fails because `conway-api-key` is not in `secrets/ovh.yaml`,
  the fix is to set `sopsFile` explicitly on the secret declaration:
  ```nix
  sops.secrets."conway-api-key" = {
    sopsFile = ../secrets/neurosys.yaml;
    owner = "automaton";
    group = "automaton";
  };
  ```
  This is cleaner than adding a dummy key to OVH secrets.
- The service is marked `autonomous: false` because it requires manual steps:
  sops key provisioning, genesis prompt editing, and manual service start.
