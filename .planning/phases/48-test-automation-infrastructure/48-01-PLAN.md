# Plan 48-01: Foundation + Core Eval Checks + Live Test Suite

## Goal

Build the two-layer test infrastructure for neurosys: (1) Nix-native eval checks
that run via `nix flake check` and catch config bugs offline, and (2) BATS live
tests that SSH to running hosts and verify service health. The result is a single
`nix flake check` for config correctness and a single `nix run .#test-live --
neurosys` for runtime health. Both produce agent-parseable output (TAP for BATS,
error messages for Nix).

This plan is self-contained and valuable on its own. It covers the highest-value
tests (Tier 1 eval + Tier 2 live) without depending on Plan 02.

## Decisions Made (from Research)

- **Two-layer architecture:** Nix eval checks (offline, fast) + BATS live tests
  (SSH, runtime). No NixOS VM tests (no KVM on Contabo).
- **BATS over pytest:** Infrastructure testing is SSH + curl + systemctl. BATS
  is native, produces TAP output, available in nixpkgs.
- **Eval checks as `runCommandNoCC` derivations:** Config properties extracted at
  eval time and asserted in shell scripts inside derivations. No runtime
  dependencies, deterministic.
- **Host-parameterized live tests:** `NEUROSYS_TEST_HOST` env var selects target.
  Tests discover which services apply to each host at runtime.
- **No KVM dependency:** All tests work on Contabo VPS (BATS via SSH, eval via
  nix evaluator). No QEMU, no NixOS VM test framework.
- **bats-support + bats-assert:** Standard BATS helper libraries from nixpkgs for
  structured assertions and error messages.

## must_haves

1. `nix flake check` runs 15+ eval checks covering: both host configs evaluate, expected services defined, expected packages present, firewall ports correct, trustedInterfaces correct, impermanence critical paths present, agentd option schema validates
2. `nix run .#test-live -- neurosys` runs 30+ BATS tests covering: systemd units active, HTTP endpoints healthy, sops secrets decrypted, Docker containers running, SSH hardening, kernel sysctl values, Tailscale connected
3. Each BATS test is one assertion with a descriptive name (agents know exactly what failed)
4. BATS failure output includes: expected value, actual value, debug hint
5. `nix flake check` still passes (existing deploy-rs checks preserved)
6. All new test files are `git add`-ed before check (flake constraint)
7. `scripts/run-tests.sh` wraps both eval and live tests with a single entry point

      #!/usr/bin/env bash
      # tests/lib/common.bash — Shared helpers for neurosys BATS live tests
      #
      # Usage: load '../lib/common' at the top of each .bats file
      # Requires: NEUROSYS_TEST_HOST env var (default: neurosys)

      # --- Configuration ---
      SSH_OPTS="-o ConnectTimeout=10 -o StrictHostKeyChecking=no -o BatchMode=yes -o LogLevel=ERROR"
      HOST="${NEUROSYS_TEST_HOST:-neurosys}"
      SSH_USER="${NEUROSYS_TEST_USER:-root}"

      # --- SSH helpers ---
      ssh_cmd() {
        ssh $SSH_OPTS "${SSH_USER}@${HOST}" "$@"
      }

      # Run a command on the remote host, capturing output
      remote() {
        ssh_cmd "$@"
      }

      # curl via SSH to the remote host (hits localhost on the remote)
      remote_curl() {
        local url="$1"; shift
        ssh_cmd "curl -sf --max-time 10 '$url'" "$@"
      }

      # --- Assertion helpers ---
      # Assert a systemd unit is active
      assert_unit_active() {
        local unit="$1"
        local result
        result="$(remote systemctl is-active "$unit" 2>&1)" || {
          echo "FAIL: unit '$unit' is not active (state: ${result:-unknown})"
          echo "DEBUG: ssh ${SSH_USER}@${HOST} journalctl -u $unit --no-pager -n 20"
          return 1
        }
        if [[ "$result" != "active" ]]; then
          echo "FAIL: unit '$unit' state is '$result', expected 'active'"
          echo "DEBUG: ssh ${SSH_USER}@${HOST} systemctl status $unit"
          return 1
        fi
      }

      # Assert an HTTP endpoint returns 200
      assert_http_ok() {
        local url="$1"
        local label="${2:-$url}"
        local status
        status="$(remote curl -so /dev/null -w '%{http_code}' --max-time 10 "$url" 2>&1)" || {
          echo "FAIL: $label — could not connect (curl exit: $?)"
          echo "DEBUG: ssh ${SSH_USER}@${HOST} curl -v '$url'"
          return 1
        }
        if [[ "$status" != "200" ]]; then
          echo "FAIL: $label — HTTP $status, expected 200"
          echo "DEBUG: ssh ${SSH_USER}@${HOST} curl -sv '$url'"
          return 1
        fi
      }

      # Assert a secret file exists with correct permissions
      assert_secret_exists() {
        local path="$1"
        local expected_owner="${2:-root}"
        remote test -f "$path" || {
          echo "FAIL: secret file '$path' does not exist"
          echo "DEBUG: ssh ${SSH_USER}@${HOST} ls -la /run/secrets/"
          return 1
        }
        local owner
        owner="$(remote stat -c '%U' "$path")"
        if [[ "$owner" != "$expected_owner" ]]; then
          echo "FAIL: secret '$path' owned by '$owner', expected '$expected_owner'"
          return 1
        fi
      }

      # Assert a sysctl value
      assert_sysctl() {
        local key="$1"
        local expected="$2"
        local actual
        actual="$(remote sysctl -n "$key" 2>&1)" || {
          echo "FAIL: sysctl '$key' not readable"
          return 1
        }
        if [[ "$actual" != "$expected" ]]; then
          echo "FAIL: sysctl $key = '$actual', expected '$expected'"
          return 1
        fi
      }

      # Check if current host is neurosys (Contabo) vs ovh
      is_neurosys() {
        [[ "$HOST" == "neurosys" ]]
      }

      is_ovh() {
        [[ "$HOST" == "neurosys-prod" ]] || [[ "$HOST" == "ovh" ]]
      }

## Risks and Mitigations

### pkgs.bats.libraries path may differ (LOW)

The `bats.libraries.bats-support` and `bats.libraries.bats-assert` attribute
paths may vary across nixpkgs versions.

**Mitigation:** Check `nix eval nixpkgs#bats.libraries` to verify the attribute
names. Fallback: use `pkgs.bats` standalone and skip bats-support/bats-assert
in favor of manual assertions in common.bash.

### Eval checks reference config paths that don't exist (MEDIUM)

The eval checks access `config.services.agentd`, `config.environment.persistence`,
etc. If these modules have different attribute paths than expected, checks will
fail with "attribute not found" errors.

**Mitigation:** Each check uses the actual module config paths from the codebase
(verified by reading modules in this planning session). If a path fails, the
check error message will clearly state which attribute is missing.

### deploy-rs checks use mapAttrs over systems (LOW)

The current `checks` output uses `builtins.mapAttrs` over deploy-rs.lib, which
may produce checks for multiple systems. The new eval checks are x86_64-linux
only.

**Mitigation:** Use `checks.${system}` directly instead of mapAttrs. The
merged result explicitly sets `checks.x86_64-linux = deployChecks // evalChecks`.

## Notes for Implementer

- All BATS test files need `#!/usr/bin/env bats` shebang and `load '../lib/common'`
- BATS `@test` names that include `$HOST` will expand at runtime, giving
  host-specific output in TAP format
- The `BATS_LIB_PATH` must be set so `bats_load_library` can find bats-support
  and bats-assert. The test-live wrapper handles this.
- Files MUST be `git add`-ed before `nix flake check` can see them
- shellcheck may warn on BATS-specific syntax (`@test`, `$BATS_*` variables).
  The shellcheck check uses `|| true` to not block on BATS syntax warnings.
- The `common.bash` `remote_curl` function runs curl on the remote host, not
  locally. This is important because services bind to localhost.
- Do NOT test secret VALUES, only existence and permissions
