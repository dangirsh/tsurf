  <action>
1. Add srvos flake input to flake.nix:
   ```
   srvos = {
     url = "github:nix-community/srvos";
     inputs.nixpkgs.follows = "nixpkgs";
   };
   ```

2. Add `srvos` to the outputs function parameters (alongside existing `self, nixpkgs, ...`).

3. Add `srvos.nixosModules.server` as the FIRST module in `nixosConfigurations.neurosys.modules` list — before `disko.nixosModules.disko`. This ensures srvos `mkDefault` values have lowest priority and all existing explicit settings win automatically.

4. Run `nix flake lock --update-input srvos` to add srvos to flake.lock.

5. In `hosts/neurosys/default.nix`, add `lib` to the function arguments (`{ config, pkgs, inputs, lib, ... }:`), then add these srvos overrides after the static IP block and before `system.stateVersion`:

   ```nix
   # --- srvos overrides ---
   # Contabo VPS uses scripted networking for static IP, not systemd-networkd
   networking.useNetworkd = lib.mkForce false;
   # Dev server: agents and humans need man pages and --help
   srvos.server.docs.enable = true;
   # Helpful for interactive sessions
   programs.command-not-found.enable = true;
   ```

   Use `lib.mkForce false` for networkd (srvos uses `mkDefault true`; without mkForce the srvos default could win). The other two use direct set which beats `mkDefault`.

   Do NOT add a `boot.initrd.systemd.enable` override — research confirmed srvos does NOT set this option.

6. Run `nix flake check` to validate. All existing explicit settings (firewall, SSH, mutableUsers, sudo, timezone, gc, configurationLimit) should automatically win over srvos `mkDefault` values.
  </action>

  </verify>
  <done>srvos server profile imported as first module, 3 overrides in hosts/neurosys/default.nix, nix flake check passes, flake.lock updated with srvos pin</done>

   ```nix

   ```
   Then use `system` and `pkgs` throughout the outputs (replacing `"x86_64-linux"` literals where applicable for DRY).

4. Add `formatter` output:
   ```nix
   formatter.${system} = treefmtEval.config.build.wrapper;
   ```

5. Add `devShells` output:
   ```nix
   devShells.${system}.default = pkgs.mkShell {
     packages = [
       pkgs.sops
       pkgs.age
       pkgs.nixfmt
       pkgs.shellcheck
       deploy-rs.packages.${system}.default
     ];
   };
   ```

6. Update the existing `checks` output to merge deploy-rs checks with the formatting check:
   ```nix
   checks = builtins.mapAttrs
     (system: deployLib: deployLib.deployChecks self.deploy // {
       formatting = treefmtEval.config.build.check self;
     })
     deploy-rs.lib;
   ```

7. Create `treefmt.nix` at the repo root:
   ```nix
   { pkgs, ... }: {
     projectRootFile = "flake.nix";
     programs.nixfmt.enable = true;
     programs.shellcheck.enable = true;
   }
   ```

8. `git add treefmt.nix` (flakes only see tracked files).

Prefer (a) if feasible; fall back to (b) if formatting changes are too large to include in this plan.
