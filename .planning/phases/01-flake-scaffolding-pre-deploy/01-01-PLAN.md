  <action>
Stage all new files for git tracking (Nix flakes only see git-tracked files), generate the lock file, and verify the complete configuration.

1. Run `git add flake.nix hosts/ modules/ home/ secrets/` to make all files visible to the flake evaluator.

2. Run `nix flake update` to fetch all inputs and generate `flake.lock`. This pins nixpkgs (nixos-25.11), home-manager (release-25.11), sops-nix, and disko to specific revisions.

3. Run `nix flake check` to verify the entire configuration evaluates without errors. This checks:
   - All flake inputs resolve
   - `nixosConfigurations.acfs` evaluates (module imports, option declarations, type checking)
   - No missing files or broken references

4. If `nix flake check` fails:
   - Read the error message carefully
   - Common issues: missing `git add` for a file, typo in import path, wrong option name
   - Fix the issue in the relevant file and re-run `nix flake check`
   - Do NOT skip errors or mark the task done with failing checks

5. Run `git add flake.lock` to track the lock file.

Note: `nix flake check` evaluates configuration structure but does NOT build the system or attempt deployment. It verifies that all Nix expressions are well-formed and all module options are valid. sops-nix secret values are NOT decrypted during evaluation — only the module structure is checked.
  </action>
