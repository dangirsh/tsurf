NixOS merges `substituters` and `trusted-public-keys` lists across modules, so this adds to the default cache.nixos.org entries in base.nix.

**3. Systemd cgroup slice for agent isolation:**
```nix
systemd.slices."agent" = {
  description = "Agent workload isolation slice";
  sliceConfig = {
    CPUWeight = 100;  # Default weight; agents share equally within this slice
  };
};
```
Per CONTEXT.md: CPU uses CPUWeight (dynamic fair-share). No memory limits for now (96GB ample).

**4. User linger for persistent systemd user instance:**
```nix
users.users.dangirsh.linger = true;
```
Per research Pitfall 5: Without linger, `systemd-run --user` fails with "Failed to connect to bus". Linger ensures the user systemd instance persists.

**5. agent-spawn launcher script** via writeShellApplication:
```nix
let
  agent-spawn = pkgs.writeShellApplication {
    name = "agent-spawn";
    runtimeInputs = with pkgs; [ tmux systemd ];
    text = ''
      NAME="''${1:?Usage: agent-spawn <name> <project-dir> [claude|codex]}"
      PROJECT_DIR="''${2:?Usage: agent-spawn <name> <project-dir> [claude|codex]}"
      AGENT="''${3:-claude}"

      case "$AGENT" in
        claude) CMD="claude" ;;
        codex)  CMD="codex" ;;
        *)      echo "Unknown agent: $AGENT (expected: claude, codex)"; exit 1 ;;
      esac

      # Validate project directory exists
      if [ ! -d "$PROJECT_DIR" ]; then
        echo "Error: Project directory does not exist: $PROJECT_DIR"
        exit 1
      fi

      # Launch in cgroup-isolated tmux session
      systemd-run --scope --slice=agent.slice \
        -p CPUWeight=100 \
        -- tmux new-session -d -s "$NAME" -c "$PROJECT_DIR" "$CMD"

      echo "Agent '$NAME' spawned in tmux session (agent.slice)"
      echo "Attach: tmux attach -t $NAME"
    '';
  };
in
```
Then add `agent-spawn` to `environment.systemPackages`.

Per research Open Question 1: Use `systemd-run --scope` (without --user) for system-level scope. This is more reliable than --user which requires user-level slice configuration. The cgroup isolation still works at system level. If `--scope` needs root, use `--user --scope` and the linger setting ensures user systemd is available.

**Decision for implementation:** Start with `systemd-run --scope --slice=agent.slice` (no --user flag). If this requires root privileges to run as dangirsh, fall back to `systemd-run --user --scope --slice=agent.slice`. Document the chosen approach with a @decision annotation.
