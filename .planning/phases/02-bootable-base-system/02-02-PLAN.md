<task type="auto">
  <name>Task 1: Pre-flight checks and deploy via nixos-anywhere</name>
  <files></files>
  <action>
Run pre-flight checks, then execute nixos-anywhere to deploy NixOS to the Contabo VPS.

**Step 1: Pre-flight validation**
```bash
# Verify flake builds cleanly
nix flake check

# Verify the pre-generated host key exists
ls -la tmp/host-key/ssh_host_ed25519_key
```

If `nix flake check` fails, STOP and fix the issue before deploying.

**Step 2: Check Contabo kexec support (optional but recommended)**
The user will need to provide the CONTABO_IP. Before deployment, optionally verify:
```bash
ssh root@<CONTABO_IP> 'cat /proc/sys/kernel/kexec_load_disabled'
# Expected: 0 (kexec enabled)
```
If kexec is disabled (value = 1), nixos-anywhere will fail. Fallback: use Contabo rescue/VNC to boot a NixOS ISO.

**Step 3: Deploy with nixos-anywhere**
```bash
# Create temp directory for --extra-files
temp=$(mktemp -d)
trap 'rm -rf "$temp"' EXIT

# Stage SSH host key with correct structure and permissions
install -d -m755 "$temp/etc/ssh"
cp tmp/host-key/ssh_host_ed25519_key "$temp/etc/ssh/ssh_host_ed25519_key"
chmod 600 "$temp/etc/ssh/ssh_host_ed25519_key"

# Deploy NixOS
nix run github:nix-community/nixos-anywhere -- \
  --extra-files "$temp" \
  --flake '.#acfs' \
  --target-host "root@<CONTABO_IP>"
```

Replace `<CONTABO_IP>` with the actual IP provided by the user.

**Expected outcome:** nixos-anywhere connects via SSH, kexecs into NixOS installer, partitions disk via disko, installs NixOS, reboots. The server comes back with the pre-generated SSH host key, allowing sops-nix to decrypt secrets.

**If nixos-anywhere hangs at kexec (>2 minutes):** Try with an older kexec image: `--kexec <url>`. If still failing, use Contabo VNC/rescue console.

**If server doesn't come back after reboot:** Use Contabo VNC console to check boot output. Likely GRUB or VirtIO issue — check boot.nix and hardware.nix.

**After deployment completes:**
```bash
# Remove stale known_hosts entry (host key changed)
ssh-keygen -R <CONTABO_IP>
```

NixOS deployed to Contabo VPS with:

  </what-built>
  <how-to-verify>
Run these verification commands from your local machine:

**1. SSH access (NET-01)**
```bash
# Should succeed with key auth
ssh dangirsh@<CONTABO_IP> 'echo "SSH key auth works"'

# Should fail — root login rejected
ssh root@<CONTABO_IP> 'echo "SHOULD NOT SEE THIS"' 2>&1
# Expected: Permission denied (publickey)

# Should fail — password auth rejected
ssh -o PreferredAuthentications=password -o PubkeyAuthentication=no dangirsh@<CONTABO_IP> 2>&1
# Expected: Permission denied (publickey)
```

**2. System identity (SYS-02)**
```bash
ssh dangirsh@<CONTABO_IP> 'hostname && timedatectl show --property=Timezone --value && locale | head -1'
# Expected: acfs / Europe/Berlin / LANG=C.UTF-8
```

**3. Firewall (NET-02, NET-04)**
From an external machine (not the server itself):
```bash
nmap -sT -p 22,80,443,22000,8080 <CONTABO_IP>
# Expected: 22 open, 80/443/22000 open or filtered (no service yet), 8080 filtered/closed
```

**4. Nix GC and store optimization (BOOT-05, BOOT-06)**
```bash
ssh dangirsh@<CONTABO_IP> 'sudo systemctl list-timers | grep nix-gc'
# Expected: nix-gc.timer listed and active

ssh dangirsh@<CONTABO_IP> 'nix show-config | grep auto-optimise-store'
# Expected: auto-optimise-store = true
```

**5. User groups (SYS-01)**
```bash
ssh dangirsh@<CONTABO_IP> 'groups'
# Expected: includes "wheel" (docker may not appear until Phase 3 enables Docker)
```

**6. sops-nix host key (BOOT-03 prerequisite)**
```bash
ssh dangirsh@<CONTABO_IP> 'ls -la /etc/ssh/ssh_host_ed25519_key'
# Expected: -rw------- (mode 600), exists
```
