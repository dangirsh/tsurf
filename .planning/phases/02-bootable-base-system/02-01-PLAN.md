  <action>
Replace the contents of `modules/networking.nix` with the hardened configuration. Three changes from the Phase 1 version:

1. **Add `networking.nftables.enable = true;`** — Switches firewall backend from iptables to nftables. This works alongside `networking.firewall.enable = true` (NixOS generates nftables rules instead of iptables rules). Satisfies NET-02.

2. **Add `KbdInteractiveAuthentication = false;`** to `services.openssh.settings` — Without this, PAM-based keyboard-interactive auth could still be attempted. Phase 1 missed this setting. Satisfies NET-01.

3. **Change `PermitRootLogin` from `"prohibit-password"` to `"no"`** — The Phase 1 value still allows root login via SSH key. NET-01 requires root login to be fully rejected. After deployment, all remote management uses `dangirsh` + sudo.

The existing `networking.firewall.allowedTCPPorts = [ 22 80 443 22000 ]` already satisfies NET-04. Keep it unchanged.

Add `@decision` annotations as file header comments documenting the requirement traceability (NET-01, NET-02, NET-04).

Final file content:
```nix
# modules/networking.nix
# @decision NET-01: key-only SSH, no root login
# @decision NET-02: default-deny nftables firewall
# @decision NET-04: ports 22, 80, 443, 22000 only
{ config, lib, pkgs, ... }: {
  # NET-02: Use nftables backend (modern replacement for iptables)
  networking.nftables.enable = true;

  # NET-02 + NET-04: Default-deny firewall with explicit allowlist
  networking.firewall = {
    enable = true;
    allowedTCPPorts = [
      22     # SSH (NET-01)
      80     # HTTP
      443    # HTTPS
      22000  # Syncthing (NET-04)
    ];
  };

  # NET-01: SSH server — key-only authentication, no root login
  services.openssh = {
    enable = true;
    settings = {
      PasswordAuthentication = false;
      KbdInteractiveAuthentication = false;
      PermitRootLogin = "no";
    };
  };
}
```

Do NOT modify flake.nix (Phase 3.1 conflict zone).
