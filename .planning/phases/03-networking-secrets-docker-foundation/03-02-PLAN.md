**Create `modules/docker.nix`** — New module for Docker engine configuration. This module declares Docker with `--iptables=false`, NAT for container outbound access, and bridge trust for container-to-container communication. The module uses NixOS option merging to add to networking config declared in networking.nix.

```nix
# modules/docker.nix
# @decision DOCK-01: Docker engine with --iptables=false, NixOS owns the firewall
{ config, pkgs, ... }: {

  # --- Docker engine ---
  virtualisation.docker = {
    enable = true;
    daemon.settings = {
      iptables = false;           # NixOS owns the firewall, not Docker
      log-driver = "journald";    # Container logs in systemd journal
    };
  };

  # --- NAT for container outbound internet access ---
  # Required because Docker --iptables=false disables Docker's own NAT/masquerade.
  # Without this, containers cannot reach the internet.
  networking.nat = {
    enable = true;
    internalInterfaces = [ "docker0" ];
    externalInterface = "eth0";   # NOTE: verify on Contabo -- may be ens3
  };

  # --- Trust Docker bridge for container-to-container traffic ---
  # NixOS merges trustedInterfaces lists across modules, so this adds
  # to the tailscale0 trust declared in networking.nix.
  # User-defined bridge networks (br-<hash>) work because
  # networking.firewall.filterForward defaults to false (FORWARD accepts all).
  networking.firewall.trustedInterfaces = [ "docker0" ];
}
```

Key design decisions:
- Docker's `iptables = false` means Docker does NOT create firewall rules. No MASQUERADE, no FORWARD, no DNAT.
- `networking.nat` provides masquerade for docker0 traffic so containers can reach the internet.
- `docker0` in `trustedInterfaces` allows container-to-host traffic on the default bridge.
- `filterForward = false` (NixOS default, NOT set here) ensures FORWARD chain accepts all traffic. This allows inter-container communication on user-defined bridges (br-<hash>).
- `externalInterface = "eth0"` — the Contabo interface name. Research notes this may be `ens3`; must be verified post-deploy. If wrong, NAT won't work and containers will have no outbound access.
- `log-driver = "journald"` — container logs go to systemd journal, not JSON files on disk.
- Docker socket is NOT exposed to containers (NixOS default).
