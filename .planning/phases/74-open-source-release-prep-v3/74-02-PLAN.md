## Risks

### git rm --cached leaves files on disk (NON-ISSUE)
`git rm --cached` removes from tracking but leaves the file on disk. This is intentional —
`.test-status` is still generated by hooks and used locally; `server.py` can be restored
from history if needed. The files just should not be in the committed tree.

### src/ gitignore may be too broad (LOW)
Adding `src/` to `.gitignore` prevents any future `src/` directory from being tracked.
If a legitimate `src/` is needed in the future, the gitignore can be narrowed to
`src/neurosys-mcp/` instead. For now, the repo has no `src/` content that should be public.

### README polish is subjective (LOW)
The README changes are minimal and functional (fix broken link, add testing section, fix typo).
No major restructuring that could be controversial.
