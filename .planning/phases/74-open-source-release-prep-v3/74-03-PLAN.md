## Objective

Reset the git history to a single clean commit. The current repo has 728 commits containing
personal identifiers throughout the history (IPs, usernames, email addresses, Cachix keys,
etc.). Even after scrubbing the current tree (Plan 74-01/74-02), `git log -p` would reveal
all prior values. A fresh orphan branch with a single commit is the cleanest approach.

**This plan is NOT autonomous** because it involves destructive git operations (orphan branch
replacing main, force push) that require explicit user confirmation before execution.

After this plan, the public repo has a single commit with a clean tree, `nix flake check`
passes, and the private overlay continues to work.

## must_haves

- A checkpoint of the current `main` branch exists as a tag or backup branch (e.g.,
  `pre-phase-74-backup`) BEFORE any destructive operation
- All tracked files from Plans 74-01 and 74-02 are present in the new commit (nothing
  lost except: `src/neurosys-mcp/server.py`, `.test-status`, `.claude/.test-status`,
  `tmp/ovh_ssh_host_ed25519_key.pub`)
- The `.planning/` directory is NOT included in the fresh commit (it is in `.gitignore`
  and contains internal project management — not relevant for public discovery)
- The `secrets/` directory IS included (it contains sops-encrypted files that are needed
  for `nix flake check` to pass)
- The `.sops.yaml` file IS included (needed for sops-nix decryption config)
- `nix flake check` passes on the fresh orphan branch
- The fresh commit uses a clean, professional commit message
- User has explicitly confirmed before the force push to `origin/main`

## Risks

### Force push destroys remote history (MITIGATED)
The backup tag `pre-phase-74-backup` is pushed to origin before the force push. Old history
is fully recoverable. Additionally, `--force-with-lease` prevents accidentally overwriting
someone else's push.

### Private overlay breaks after history reset (LOW)
The private overlay's `neurosys` flake input tracks a specific commit hash in its lockfile.
After the history reset, `nix flake update neurosys` in the private overlay will resolve
to the new single commit. No module renames or removals were made, so all imports should
resolve. The only changes are content (identifier scrub) and the `nix-secret-proxy` URL
(which the private overlay may override via `follows` anyway).

### CI/CD references old commits (LOW)
If any CI workflow references specific commit SHAs, those will break. The `.github/workflows/test.yml`
should only reference branch names (`main`), not specific SHAs. Verify after push.

### Orphan branch missing files (MEDIUM)
`git checkout --orphan` + `git add .` relies on `.gitignore` to exclude the right files.
If `.gitignore` is wrong, files could be missing or extra files included. Mitigation:
Task 74-03-C includes explicit verification of what is staged, and Task 74-03-D runs
`nix flake check` which catches missing files.
