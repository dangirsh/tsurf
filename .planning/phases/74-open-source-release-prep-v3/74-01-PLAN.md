## Risks

### nix-secret-proxy GitHub URL may be private or have diverged (MEDIUM — MITIGATED)
Task 74-01-A now checks accessibility with `curl` before changing the URL. If the GitHub repo
returns HTTP 404 (private or non-existent), the `path:` input is kept with an explanatory comment
and the flake.lock is left unchanged. If accessible (HTTP 200), the URL is changed and
`nix flake update nix-secret-proxy` fetches the latest commit. Either path preserves a working
`nix flake check`.

### Cachix placeholder key breaks nix evaluation (LOW)
NixOS `extra-trusted-public-keys` are strings — they are not validated at eval time. The placeholder
will only cause issues at runtime (nix daemon refuses to trust it). This is acceptable for a public
template; the private overlay replaces the key.

### Deploy script placeholders break deploy.sh (NON-ISSUE by design)
The public repo's `deploy.sh` already hard-refuses to deploy (safety guard on line 210). The
placeholder IPs would only matter if someone disabled that guard, at which point they have already
taken responsibility for the config.
