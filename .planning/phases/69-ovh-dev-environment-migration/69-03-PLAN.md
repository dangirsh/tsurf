## Risks

### Live test infrastructure may not support neurosys-dev host (LOW)
The live test runner needs SSH access to neurosys-dev. If the test infrastructure only
targets `neurosys`, the `--host neurosys-dev` flag may not work out of the box. Check
the `scripts/run-tests.sh` and `tests/lib/common.bash` for host resolution logic.

### is_ovh helper may need different hostname matching (LOW)
OVH hostname is `neurosys-dev` in Tailscale but could be `neurosys-prod` in system
hostname. The `is_ovh` helper should match both to be safe. Check how `is_neurosys`
is implemented and follow the same pattern.

### Public repo flake check doesn't test OVH secret-proxy (NON-ISSUE)
The public repo's eval checks test the public template config which has no secret-proxy
services declared. OVH-specific eval checks are in the private overlay's
`private-checks.nix`. This is correct by design.
