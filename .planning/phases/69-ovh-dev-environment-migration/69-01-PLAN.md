## Risks

### repos.nix removal breaks activation script merge (NON-ISSUE)
Since repos.nix is removed from commonModules, and each host gets only its own inline
module via contaboModules or ovhModules, there is exactly one
`system.activationScripts.clone-repos` definition per host. NixOS merges activation
scripts by name using string concatenation, but with a single definition there is
nothing to merge. No conflict is possible.

### anthropic-api-key sops edit fails (LOW)
If the admin age key is unavailable or the `.sops.yaml` creation_rules don't match,
`sops --set` will fail. The admin key should be available on the local machine (acfs).
If not, the user must provide the key or use `SOPS_AGE_KEY_FILE` to point to it.

### worldcoin/ai requires PAT with org access (MEDIUM)
The `worldcoin/ai` repo requires the `github-pat` to have access to the `worldcoin`
GitHub organization. If the PAT lacks org access, only this one clone will fail (with a
non-fatal warning). All other repos will succeed. The user can add org scope to the PAT
later if needed.
