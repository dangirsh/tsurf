## Objective

Deploy the updated private overlay config to OVH VPS (neurosys-dev) and validate that all
components are functioning: secret-proxy-dev service, repo cloning, agent sandbox with
API key injection, and Syncthing. The acceptance test is a successful sandboxed Claude Code
session on OVH — this is the key success criterion for the entire phase.

This plan is NOT autonomous because it involves SSH to a live server and manual verification
of a sandboxed Claude Code session.

## must_haves

- `secret-proxy-dev.service` is active on OVH
- Secret-proxy responds on `http://localhost:9091/` on OVH (any HTTP status, not connection refused)
- All 13 repos are cloned to `/data/projects/` on OVH
- `anthropic-api-key` secret is decrypted at `/run/secrets/anthropic-api-key` on OVH
- A sandboxed Claude Code session starts successfully on OVH with API access via secret-proxy
- Deploy health checks updated to include `secret-proxy-dev` for OVH node

## Risks

### Deploy rollback due to secret-proxy start failure (MEDIUM)
If the anthropic-api-key sops secret fails to decrypt on OVH (wrong age key, corrupt file),
the secret-proxy-dev service will fail to start, and deploy.sh health checks will fail.
deploy-rs magic rollback will revert to the previous config. Fix: verify sops decryption
before deploying by checking the age key path in OVH config.

### Repo cloning fails due to github-pat placeholder (MEDIUM)
If `github-pat` in `secrets/ovh.yaml` is also a placeholder, all private repo clones will
fail. The activation script logs warnings but continues. Fix: populate github-pat in
secrets/ovh.yaml alongside anthropic-api-key (Task 69-01-C).

### agent-spawn not configured for secret-proxy env vars (LOW)
The bwrap sandbox wrapper (`agent-spawn`) may not automatically pick up the secret-proxy
env vars. If so, the acceptance test (Task E) can still pass by manually setting the env
vars. A follow-up task to wire bwrapArgs into agent-spawn can be added if needed.

### OVH host unreachable via Tailscale MagicDNS (LOW)
If Tailscale is down or the hostname `neurosys-dev` doesn't resolve, use the public IP
`135.125.196.143` or Tailscale IP `100.113.72.9` directly.
