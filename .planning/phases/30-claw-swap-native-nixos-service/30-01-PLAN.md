# Plan 30-01: Module Rewrite + Flake Check

## Goal

Rewrite the claw-swap Nix module to replace Docker containers with native
NixOS services (`services.postgresql` + a systemd service for the Node.js
app). Update three neurosys support modules (impermanence, restic, homepage).
Validate with `nix flake check` in both repos. No deployment in this plan.

## Decisions Made (from Research)

- **DB auth**: Unix socket trust. No password in `DATABASE_URL`. The OS user
  `claw-swap` accesses the DB socket via `local trust` in `pg_hba.conf`.
  The `claw-swap-db-password` secret becomes unused and is removed from the
  module (the encrypted key in `secrets/claw-swap.yaml` stays until confirmed
  safe to rotate; removing it from the module is enough for this plan).
- **Port binding**: Unix socket only (`postgres:///claw_swap?host=/run/postgresql`).
  No `enableTCPIP` needed.
- **`claw-swap-app.nix` refactor**: The file currently returns a Docker layered
  image. Change it to return the `buildNpmPackage` derivation directly.
  Update the flake `packages.claw-swap-app` output to match.
- **Module location**: Keep `claw-swap` in `commonModules` (both hosts get
  `services.postgresql`). The Contabo host gets an idle postgres instance;
  acceptable. Moving to OVH-only is a follow-up simplification.
- **`/run/postgresql` access**: The `claw-swap` system user needs access to
  the Unix socket at `/run/postgresql`. NixOS sets `/run/postgresql` to mode
  2775 owned by `postgres:postgres`. Add `claw-swap` to the `postgres` group
  so the socket is accessible without broadening pg_hba trust.

## must_haves

1. `nix flake check` passes in `/data/projects/claw-swap`
2. `nix flake check` passes in `/data/projects/neurosys`
3. No Docker containers or Docker network references remain in `nix/module.nix`

8. `modules/homepage.nix` uses `siteMonitor` for claw-swap (not `container`)
9. `pg_hba.conf` has NO `local claw_swap claw peer` rule (only trust rule); `@decision CSVC-01` annotation present
10. `modules/docker.nix` DOCK-03 annotation updated to remove claw-swap reference

      2. System user and group:
         ```nix
         users.users.claw-swap = {
           isSystemUser = true;
           group = "claw-swap";
           extraGroups = [ "postgres" ];
         };
         users.groups.claw-swap = {};
         ```
         The `extraGroups = [ "postgres" ]` gives the service user access to
         the `/run/postgresql` Unix socket (mode 2775, group `postgres`).

      3. PostgreSQL service:
         ```nix
         services.postgresql = {
           enable = true;
           package = pkgs.postgresql_16;
           ensureDatabases = [ "claw_swap" ];
           ensureUsers = [{
             name = "claw";
             ensureDBOwnership = true;
           }];
           authentication = lib.mkOverride 10 ''
             local all postgres peer
             local claw_swap claw trust
           '';
         };
         ```
         The `local claw_swap claw trust` rule allows the `claw-swap` OS user
         (which is NOT named `claw`) to connect as DB user `claw` via the
         Unix socket without a password.

         **IMPORTANT**: Do NOT add a `local claw_swap claw peer` rule above
         the trust rule. PostgreSQL evaluates pg_hba.conf rules in order and
         stops (rejects) at the first matching rule whose auth method fails.
         Peer auth requires OS username == DB username (`claw-swap` != `claw`),
         so a peer rule would reject the connection before trust is reached.

         Add a `@decision` annotation above the postgresql block:
         ```nix
         # @decision CSVC-01: pg_hba uses local trust for OS user claw-swap → DB user claw.
         # @rationale: OS user "claw-swap" != DB user "claw" so peer auth cannot be used.
         #   Trust on Unix socket is safe on this single-tenant server (no untrusted local users).
         ```

      4. Update `sops.templates."claw-swap-app-env"`:
         - Change `DATABASE_URL` to:
           `DATABASE_URL=postgres:///claw_swap?host=/run/postgresql&user=claw`
         - Change `restartUnits` on all app secrets from
           `"docker-claw-swap-app.service"` to `"claw-swap-app.service"`.

      5. Systemd service for the app:

         systemd.services.claw-swap-app = {
           description = "Claw-Swap Hono API server";

               "R2_BUCKET_NAME=claw-swap-photos"

             RestartSec = "5s";
