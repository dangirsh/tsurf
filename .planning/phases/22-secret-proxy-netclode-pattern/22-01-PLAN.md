Purpose: The proxy is the core security primitive. It ensures real API keys never enter agent sandboxes. **packages/secret-proxy/main.go** — full MITM TLS proxy (~300 lines):

The proxy must implement the following behavior precisely:

**Constants and config:**
```go
const listenAddr = "127.0.0.1:7979"

// placeholder → (real-key-path, allowed-upstream-host)
var placeholderConfig = map[string]struct {
    secretPath   string
    allowedHost  string
}{
    "PLACEHOLDER_ANTHROPIC": {"/run/secrets/anthropic-api-key", "api.anthropic.com"},
    "PLACEHOLDER_OPENAI":    {"/run/secrets/openai-api-key", "api.openai.com"},
    "PLACEHOLDER_GITHUB":    {"/run/secrets/github-pat", "api.github.com"},
}
```

**Startup sequence (main function):**
1. Generate ephemeral CA: 4096-bit RSA key (or P-256 ECDSA for speed), self-signed X.509 cert with 24h validity, CN="secret-proxy-ca", KeyUsageCertSign|KeyUsageCRLSign, IsCA=true, BasicConstraints critical.
2. Write CA cert PEM to `/run/secret-proxy-ca/ca.crt` (create dir with os.MkdirAll 0755, write file with 0644 permissions).
3. Load real API keys from secret paths into an in-memory map `realKeys map[string]string` — read each path, trim whitespace. Log a warning if a path is missing (non-fatal; injection simply won't replace that placeholder).
4. Read `PROXY_TOKEN` env var (set by the NixOS service — a static token that agent-spawn must present via Basic auth in the HTTPS_PROXY URL). If empty, log fatal and exit.
5. Listen on `127.0.0.1:7979` via `net.Listen("tcp", listenAddr)`.
6. Log "secret-proxy listening on 127.0.0.1:7979" to stderr.
7. Accept connections in a loop, handle each in a goroutine.

**Connection handler — handleConnect:**
Each new TCP connection is an HTTP/1.1 CONNECT request from the agent (Go's http.ReadRequest).

Parse the CONNECT request:
- Read the HTTP request using `bufio.NewReader` + `http.ReadRequest`
- If method is not CONNECT, send 400 Bad Request and close.
- Extract the target host:port from `req.URL.Host` (or `req.Host`).

Proxy-Authorization check (Basic auth from CONNECT request):
- Read `Proxy-Authorization` header from the CONNECT request.
- HTTP clients embed credentials in the proxy URL (http://token:${PROXY_TOKEN}@127.0.0.1:7979) and automatically send `Proxy-Authorization: Basic <base64(token:PROXY_TOKEN)>` with CONNECT requests. This is standard behavior for curl, Node.js undici, and Python requests.
- Decode the base64 value. Extract the password (the part after the first colon in the decoded `user:password` string).
- Compare the extracted password against the expected PROXY_TOKEN using `subtle.ConstantTimeCompare` to prevent timing attacks.
- If header is missing or token doesn't match, send 407 Proxy Auth Required and close the connection. Log the rejection with the remote address. Do NOT proceed.

Determine injection mode:
- Extract hostname (strip port from host:port).
- Check if the hostname matches any `allowedHost` in placeholderConfig.
- If yes: record which placeholder this connection is for (e.g., "PLACEHOLDER_ANTHROPIC") and the expected host.
- If no: transparent tunnel mode (no injection).

Send 200 Connection Established to client:
```
HTTP/1.1 200 Connection established\r\n\r\n
```

**Transparent tunnel (non-injection path):**
If hostname doesn't match any placeholder's allowedHost:
- Dial the target host:port directly via `net.Dial("tcp", host)`.
- Bidirectional copy: `go io.Copy(upstream, client)` and `io.Copy(client, upstream)`.
- Return when both goroutines finish.

**MITM injection path:**
For matching placeholder hosts:

1. Look up the placeholder → get realKey from in-memory map. If empty (secret file was missing at startup), close the connection with a 502-style TCP reset and log an error. Do NOT forward a placeholder to upstream.

2. Generate a per-connection TLS leaf certificate:
   - P-256 ECDSA key (fast, new per connection)
   - X.509 cert signed by the ephemeral CA
   - SAN: DNSNames = []string{hostname}
   - Validity: 1 hour
   - KeyUsage: KeyUsageDigitalSignature, ExtKeyUsage: ExtKeyUsageServerAuth

3. Wrap the client-side TCP connection in TLS using the leaf cert:
   ```go
   tlsClient := tls.Server(clientConn, &tls.Config{
       Certificates: []tls.Certificate{leafCert},
   })
   ```
   Perform the TLS handshake with the client.

4. Dial the real upstream host:port with TLS:
   ```go
   tlsUpstream, err := tls.Dial("tcp", targetHostPort, &tls.Config{
       ServerName: hostname,
   })
   ```
   If dial fails, send a TCP error to client and log.

5. **MITM request-response loop** (handles HTTP/1.1 keep-alive — multiple requests per connection):

   ```go
   clientReader := bufio.NewReader(tlsClient)
   upstreamReader := bufio.NewReader(tlsUpstream)
   for {
       req, err := http.ReadRequest(clientReader)
       if err != nil { break }

       // Cross-placeholder validation: ensure any placeholder in the Authorization
       // header matches the placeholder expected for this connection's target host.
       // Prevents an agent from sending PLACEHOLDER_OPENAI over a CONNECT tunnel to
       // api.anthropic.com, which would cause the Anthropic real key to be injected
       // and forwarded to the wrong service (cross-service key leakage).
       authHeader := req.Header.Get("Authorization")
       for otherPlaceholder := range placeholderConfig {
           if otherPlaceholder != expectedPlaceholder && strings.Contains(authHeader, otherPlaceholder) {
               log.Printf("cross-placeholder violation: connection for %s received %s in Authorization — closing", expectedPlaceholder, otherPlaceholder)
               tlsClient.Close()
               tlsUpstream.Close()
               return
           }
       }

       // Header injection: replace expected placeholder with real key only.
       if strings.Contains(authHeader, expectedPlaceholder) {
           req.Header.Set("Authorization", strings.ReplaceAll(authHeader, expectedPlaceholder, realKey))
           log.Printf("injected real key for %s (placeholder replaced)", expectedPlaceholder)
       }

       // Allowlist enforcement: verify request Host header matches expected host.
       reqHost := req.Host
       if reqHost == "" {
           reqHost = req.URL.Host
       }
       reqHostname := strings.Split(reqHost, ":")[0]
       if reqHostname != "" && reqHostname != expectedHost {
           log.Printf("allowlist violation: %s session sent request to %s — closing", expectedPlaceholder, reqHostname)
           tlsClient.Close()
           tlsUpstream.Close()
           return
       }

       // Forward modified request to upstream.
       if err := req.Write(tlsUpstream); err != nil { break }

       // Read response from upstream and forward to client.
       resp, err := http.ReadResponse(upstreamReader, req)
       if err != nil { break }
       if err := resp.Write(tlsClient); err != nil {
           resp.Body.Close()
           break
       }
       resp.Body.Close()

       // Respect connection close semantics (HTTP/1.0 or Connection: close).
       if req.Close || resp.Close { break }
   }
   tlsClient.Close()
   tlsUpstream.Close()
   ```

   Where `expectedPlaceholder` and `expectedHost` are determined from the `placeholderConfig` lookup before the loop begins (when the CONNECT target host was matched).

**Logging:** All log lines go to stderr via `log.Printf`. Log: startup, CONNECT received (host, placeholder match or transparent), auth failures, injection events (placeholder replaced — do NOT log the real key value), allowlist violations, cross-placeholder violations, upstream dial errors. Never log Authorization header values or secret key contents.

**Important implementation notes:**
- Use `crypto/tls`, `crypto/x509`, `crypto/ecdsa`, `crypto/elliptic`, `crypto/rand`, `crypto/subtle`, `math/big`, `encoding/pem`, `encoding/base64` from stdlib — no external deps.
- `http.ReadRequest` expects a reader, not a full connection. Use `bufio.NewReader(conn)` for reading. For writing responses back to the client before the TLS handshake, write directly to the raw `net.Conn`.
- After the 200 Connection Established response, the client immediately starts TLS. The raw conn must be wrapped with `tls.Server` before any further reads.
- The leaf cert generation is safe to do inline per-connection (P-256 key gen is ~1ms).
- Create `bufio.NewReader` once per connection and reuse across loop iterations — do NOT create a new reader per request, as that would discard buffered data from the previous read.
- `req.Write(upstream)` handles the body automatically. `http.ReadResponse` followed by `resp.Write(client)` handles the full response including body. Always call `resp.Body.Close()` to drain the body and allow the connection to handle the next request.
