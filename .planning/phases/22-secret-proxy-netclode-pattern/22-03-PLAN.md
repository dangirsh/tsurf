<task type="auto">
  <name>Task 1: Add proxy-token to sops-encrypted secrets file</name>
  <files>secrets/neurosys.yaml</files>
  <action>
Before deploying, the `proxy-token` secret must be added to `secrets/neurosys.yaml`. This is the only secret that needs to be provisioned — it is a random string used to authenticate agent-spawn to the proxy service.

Generate a random token:
```bash
openssl rand -hex 32
```

Add it to the sops-encrypted file:
```bash
# From project root (where .sops.yaml lives)
sops secrets/neurosys.yaml
```

In the editor, add the key:
```yaml
proxy-token: "<generated-hex-string>"
```

Save and exit. sops will re-encrypt the file with the configured age keys.

After editing, verify the secret is present (without decrypting):
```bash
grep 'proxy-token' secrets/neurosys.yaml
```
You should see an encrypted ciphertext entry (not the plaintext value).

**Commit the updated secrets file:**
```bash
git add secrets/neurosys.yaml
git commit -m "feat(22-secret-proxy): add proxy-token secret to neurosys.yaml"
```

Note: This is the only commit that touches secrets/neurosys.yaml. The encrypted value is safe to commit — sops encrypts it with the age key from the VPS host key.
  </action>
  <verify>
`grep 'proxy-token' secrets/neurosys.yaml` returns a line with an ENC value (ciphertext). `git diff HEAD secrets/neurosys.yaml` shows no unencrypted values.
  </verify>
  <done>
- `proxy-token` entry exists in secrets/neurosys.yaml (encrypted)
- secrets/neurosys.yaml committed with the new secret
  </done>

Inside the sandbox:

```bash
# Security invariant: real key MUST NOT be present
printenv | grep -i 'sk-ant\|sk-proj\|ghp_\|PLACEHOLDER'
# Expected: lines showing PLACEHOLDER_ANTHROPIC, PLACEHOLDER_OPENAI, PLACEHOLDER_GITHUB
# NO lines should contain real key values (sk-ant-..., sk-proj-..., ghp_...)

# Proxy env vars present with credentials in URL:

# Expected: HTTPS_PROXY=http://token:<token-value>@127.0.0.1:7979

# Expected: NODE_EXTRA_CA_CERTS=/run/secret-proxy-ca/ca.crt

# Expected: file exists (read-only bind mount)

# Secrets dir still hidden:
ls /run/secrets/ 2>&1
# Expected: No such file or directory
```

**Step 5: Verify a real API call routes through proxy successfully:**
```bash
# curl respects HTTPS_PROXY env var. Credentials in the URL are sent as Proxy-Authorization: Basic.
# Use --cacert to trust the ephemeral CA cert.
curl --cacert /run/secret-proxy-ca/ca.crt \
  -H "Authorization: Bearer PLACEHOLDER_ANTHROPIC" \
  https://api.anthropic.com/v1/models -s -o /dev/null -w "%{http_code}"
# Expected: 200 or 401 (either means proxy forwarded the request — 401 means Anthropic got
# a real key but rejected it for another reason, 200 means full success)
# NOT expected: 407 (proxy auth failure), 502 (proxy error), connection refused
```

If Claude Code itself is available, run a minimal test:
```bash
# Quick claude test (timeout 30s to avoid full session)
timeout 30 claude --print "echo hello" 2>&1 | head -20
# Expected: completes without TLS/proxy errors
```

**Step 6: Security tests (these must FAIL/BLOCK):**

```bash
# Test 1: Request with wrong proxy credentials should be blocked (407)
# Directly specify a wrong token in the proxy URL.
PROXY_TOKEN_WRONG="wrongtoken"
curl -s -o /dev/null -w "%{http_code}" \
  --cacert /run/secret-proxy-ca/ca.crt \
  -x "http://token:${PROXY_TOKEN_WRONG}@127.0.0.1:7979" \
  -H "Authorization: Bearer PLACEHOLDER_ANTHROPIC" \
  https://api.anthropic.com/v1/models
# Expected: 407 (proxy authentication required — wrong credentials)

# Test 2: PLACEHOLDER_ANTHROPIC session should NOT reach openai.com
# (Allowlist violation — proxy should block or return error)
curl --cacert /run/secret-proxy-ca/ca.crt \
  -H "Authorization: Bearer PLACEHOLDER_ANTHROPIC" \
  https://api.openai.com/v1/models -v -s -o /dev/null 2>&1 | grep -E 'HTTP|< HTTP|allowi|blocked|502'
# Expected: connection reset or 502 (allowlist violation — PLACEHOLDER_ANTHROPIC
# is not allowed to reach openai.com)

# Test 3: Cross-placeholder injection blocked
# Send PLACEHOLDER_OPENAI header over a CONNECT tunnel to api.anthropic.com.
# The proxy should detect the mismatch and close the connection.
curl --cacert /run/secret-proxy-ca/ca.crt \
  -H "Authorization: Bearer PLACEHOLDER_OPENAI" \
  https://api.anthropic.com/v1/models -v -s -o /dev/null 2>&1 | grep -E 'HTTP|cross|closed'
# Expected: connection closed (cross-placeholder violation)
```

**Step 7: Check proxy logs for injection events:**
```bash
# From outside sandbox (different terminal as dangirsh)
journalctl -u secret-proxy -n 30
# Should show: CONNECT received, placeholder replaced (no key values in log),
# any allowlist violations from Step 6
```
