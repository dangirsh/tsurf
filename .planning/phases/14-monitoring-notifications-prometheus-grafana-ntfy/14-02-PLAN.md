  <name>Task 1: Add ntfy notifications to deploy.sh</name>
  <files>scripts/deploy.sh</files>
  <action>
Add ntfy notification calls to `scripts/deploy.sh` at both success and failure exit points.

**Key design decision:** deploy.sh runs on the LOCAL machine, not the server. ntfy listens on the SERVER. The notification must target the server via SSH (run curl on the server) since ntfy is Tailscale-only and the local machine may not have direct access to the ntfy port. Use SSH to the target host to send the curl.

Add a `NTFY_TOPIC` variable near the top of the script:
```bash
NTFY_TOPIC="deploys"
```

**After the success block** (after "=== Deploy SUCCESS ==="), add:
```bash
# Best-effort ntfy notification (never blocks deploy)
ssh "$TARGET" "curl --silent --max-time 10 \
  -H 'Title: Deploy succeeded' \
  -H 'Priority: low' \
  -H 'Tags: white_check_mark' \
  -d 'acfs deployed (parts: $PARTS_REV_SHORT) in $((DURATION / 60))m $((DURATION % 60))s' \
  http://localhost:2586/$NTFY_TOPIC" 2>/dev/null || true
```

**After the failure block** (after "=== Deploy FAILED ===" and before `exit 1`), add:
```bash
# Best-effort ntfy notification (never blocks deploy)
ssh "$TARGET" "curl --silent --max-time 10 \
  -H 'Title: Deploy FAILED' \
  -H 'Priority: high' \
  -H 'Tags: rotating_light' \
  -d 'acfs deploy failed after $((DURATION / 60))m $((DURATION % 60))s. Check containers.' \
  http://localhost:2586/$NTFY_TOPIC" 2>/dev/null || true
```

IMPORTANT:
- `|| true` ensures ntfy failure never causes the deploy script to fail (ntfy might not be running yet on first deploy of this config)
- `2>/dev/null` suppresses SSH connection errors
- Use single quotes around curl headers inside the SSH command (the outer command uses double quotes for variable expansion of `$PARTS_REV_SHORT` and `$DURATION`)
- The SSH target is `$TARGET` which is already defined (default: `root@acfs`)

Note about variable expansion: `$PARTS_REV_SHORT` and `$DURATION` are local variables that need to expand on the LOCAL side. Since the SSH command is in double quotes, they will expand locally before being sent. This is the correct behavior.

<task type="auto">
  <name>Task 2: Create scripts/notify.sh -- generic ntfy notification helper</name>
  <files>scripts/notify.sh</files>
  <action>
Create `scripts/notify.sh` -- a generic helper script that any process on the server can call to send ntfy notifications. This addresses the "agent completion" notification requirement from the phase goal and provides the hookup mechanism for future restic backup notifications (Phase 7).

The script runs ON THE SERVER (localhost access to ntfy), unlike deploy.sh which runs locally and SSHes in.

```bash
#!/usr/bin/env bash
# scripts/notify.sh -- Send ntfy notification from the server
# Usage: notify.sh <topic> <title> [message] [--priority <level>] [--tags <tags>]
#
# Examples:
#   notify.sh agents "Agent completed" "Task X finished successfully"
#   notify.sh backups "Backup failed" "restic snapshot failed" --priority high --tags warning
#   notify.sh agents "Agent completed" --priority low --tags white_check_mark
#
# Designed to be called by:
#   - Agent workflows (agent-spawn exit hooks, completion scripts)
#   - Restic backup hooks (onFinish/onError) when Phase 7 is implemented
#   - Cron jobs, systemd timers, or any server-side process

set -euo pipefail

NTFY_URL="http://localhost:2586"

# Parse required args
TOPIC="${1:?Usage: notify.sh <topic> <title> [message] [--priority level] [--tags tags]}"
TITLE="${2:?Usage: notify.sh <topic> <title> [message] [--priority level] [--tags tags]}"
shift 2

# Parse optional message (first non-flag arg)
MESSAGE=""
PRIORITY="default"
TAGS=""

while [[ $# -gt 0 ]]; do
  case "$1" in
    --priority) PRIORITY="$2"; shift 2 ;;
    --tags) TAGS="$2"; shift 2 ;;
    *)
      if [[ -z "$MESSAGE" ]]; then
        MESSAGE="$1"; shift
      else
        echo "Unknown argument: $1" >&2; exit 1
      fi
      ;;
  esac
done

# Build curl args
CURL_ARGS=(
  --silent --show-error
  --max-time 10
  -H "Title: $TITLE"
  -H "Priority: $PRIORITY"
)

[[ -n "$TAGS" ]] && CURL_ARGS+=(-H "Tags: $TAGS")
[[ -n "$MESSAGE" ]] && CURL_ARGS+=(-d "$MESSAGE")

curl "${CURL_ARGS[@]}" "$NTFY_URL/$TOPIC"
```

Make executable: `chmod +x scripts/notify.sh`

Design decisions:
- Runs on the server (localhost), not via SSH like deploy.sh
- No `|| true` -- callers decide error handling (deploy.sh adds it, restic hooks may want to propagate)
- Positional args for topic + title (most common), flags for optional priority/tags
- No auth tokens needed because ntfy uses write-only default access (Tailscale-only)

scripts/notify.sh exists and is executable. Accepts topic, title, optional message, optional --priority and --tags flags. Curls localhost:2586 (ntfy on server). Can be called by agent workflows, future restic hooks, cron jobs, or any server-side process. No auth tokens required (ntfy is write-only, Tailscale-only).
