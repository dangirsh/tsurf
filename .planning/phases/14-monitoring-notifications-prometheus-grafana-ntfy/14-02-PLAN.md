  <name>Task 1: Add ntfy notifications to deploy.sh</name>
  <files>scripts/deploy.sh</files>
  <action>
Add ntfy notification calls to `scripts/deploy.sh` at both success and failure exit points.

**Key design decision:** deploy.sh runs on the LOCAL machine, not the server. ntfy listens on the SERVER. The notification must target the server via SSH (run curl on the server) since ntfy is Tailscale-only and the local machine may not have direct access to the ntfy port. Use SSH to the target host to send the curl.

Add a `NTFY_TOPIC` variable near the top of the script:
```bash
NTFY_TOPIC="deploys"
```

**After the success block** (after "=== Deploy SUCCESS ==="), add:
```bash
# Best-effort ntfy notification (never blocks deploy)
ssh "$TARGET" "curl --silent --max-time 10 \
  -H 'Title: Deploy succeeded' \
  -H 'Priority: low' \
  -H 'Tags: white_check_mark' \
  -d 'acfs deployed (parts: $PARTS_REV_SHORT) in $((DURATION / 60))m $((DURATION % 60))s' \
  http://localhost:2586/$NTFY_TOPIC" 2>/dev/null || true
```

**After the failure block** (after "=== Deploy FAILED ===" and before `exit 1`), add:
```bash
# Best-effort ntfy notification (never blocks deploy)
ssh "$TARGET" "curl --silent --max-time 10 \
  -H 'Title: Deploy FAILED' \
  -H 'Priority: high' \
  -H 'Tags: rotating_light' \
  -d 'acfs deploy failed after $((DURATION / 60))m $((DURATION % 60))s. Check containers.' \
  http://localhost:2586/$NTFY_TOPIC" 2>/dev/null || true
```

IMPORTANT:
- `|| true` ensures ntfy failure never causes the deploy script to fail (ntfy might not be running yet on first deploy of this config)
- `2>/dev/null` suppresses SSH connection errors
- Use single quotes around curl headers inside the SSH command (the outer command uses double quotes for variable expansion of `$PARTS_REV_SHORT` and `$DURATION`)
- The SSH target is `$TARGET` which is already defined (default: `root@acfs`)

Note about variable expansion: `$PARTS_REV_SHORT` and `$DURATION` are local variables that need to expand on the LOCAL side. Since the SSH command is in double quotes, they will expand locally before being sent. This is the correct behavior.
