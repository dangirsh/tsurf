Create `modules/monitoring.nix` with the following components. Use the proven module pattern from this codebase (see docker.nix, networking.nix): `{ config, pkgs, ... }:` header, `@decision` annotations at the top.

**node_exporter:**
- `services.prometheus.exporters.node.enable = true`
- Port 9100
- `enabledCollectors = [ "systemd" "processes" "tcpstat" ]`

**Prometheus:**
- `services.prometheus.enable = true`
- Port 9090
- `retentionTime = "90d"`
- `globalConfig.scrape_interval = "15s"`
- Two scrapeConfigs: `node` (localhost:node_exporter_port) and `prometheus` (localhost:prometheus_port). Use `toString config.services.prometheus.exporters.node.port` and `toString config.services.prometheus.port` for port references.
- `alertmanagers` pointing to localhost:9093
- `ruleFiles` with a single `pkgs.writeText "alert-rules.json"` containing 6 rules as Nix attrsets converted via `builtins.toJSON`:
  1. `InstanceDown` — `up < 1` for 2m, severity critical
  2. `DiskSpaceCritical` — filesystem avail < 10% on `/` for 5m, severity critical
  3. `DiskSpaceWarning` — filesystem avail < 20% on `/` for 10m, severity warning
  4. `HighMemoryUsage` — MemAvailable < 10% for 5m, severity critical
  5. `HighCpuUsage` — idle < 10% (i.e., usage > 90%) for 10m, severity warning
  6. `SystemdUnitFailed` — `node_systemd_unit_state{state="failed"} == 1` for 1m, severity critical
- Each rule should have `annotations.summary` with template variables (`{{ $labels.instance }}`)

**Alertmanager:**
- `services.prometheus.alertmanager.enable = true`
- Port 9093
- `configuration.route`: default receiver "ntfy", `group_wait = "30s"`, `group_interval = "5m"`, `repeat_interval = "4h"`. Add a route for `severity = "critical"` with `repeat_interval = "1h"`.
- Single receiver "ntfy" with `webhook_configs` URL `http://127.0.0.1:8000/hook`

**alertmanager-ntfy bridge:**
- `services.prometheus.alertmanager-ntfy.enable = true`
- `settings.http.addr = "127.0.0.1:8000"`
- `settings.ntfy.baseurl = "http://localhost:2586"`
- `settings.ntfy.notification.topic = "alerts"`
- Priority: high for firing, default for resolved (use conditional expression from research)
- Tags: rotating_light for firing, +1 for resolved
- Templates: title includes "Resolved:" prefix when resolved, description from annotations

**systemd OnFailure template:**
- `systemd.services."ntfy-failure@"` — oneshot service, scriptArgs = "%i"
- `path = [ pkgs.curl pkgs.hostname ]`
- Script: curl POST to `http://localhost:2586/alerts` with Title "Service crashed: $1", Priority "urgent", Tags "skull", body "$1 failed on $(hostname) at $(date -Iseconds)"
- Attach OnFailure to critical services: `systemd.services.docker.unitConfig.OnFailure`, `systemd.services.prometheus.unitConfig.OnFailure`, `systemd.services.grafana.unitConfig.OnFailure`

Use double-single-quotes for PromQL expressions containing special characters (standard Nix string escaping).

# fail2ban ntfy notification on ban
environment.etc."fail2ban/action.d/ntfy.local".text = ''
  [Definition]
  norestored = true
  actionban = ${pkgs.curl}/bin/curl \
    --fail --show-error --silent \
    --max-time 10 --retry 3 \
    -H "Title: fail2ban: <ip> banned" \
    -H "Priority: default" \
    -H "Tags: police_car_light" \
