**1. Rewrite agent-spawn writeShellApplication:**

The new agent-spawn script must:

a. Parse args: `<name> <project-dir> [claude|codex] [--no-sandbox] [--show-policy]`
   - First two positional args required (name, project-dir)
   - Third positional arg optional (agent type, default "claude")
   - `--no-sandbox` and `--show-policy` can appear anywhere after the first two positional args
   - Validate project dir exists and resolve with `realpath`

b. `--show-policy` mode: Print sandbox policy (project dir, visible paths, hidden paths, limits, Podman status) and exit 0.

c. Read API keys from sops secrets BEFORE entering sandbox:
   ```bash
   ANTHROPIC_KEY="$(cat /run/secrets/anthropic-api-key 2>/dev/null || true)"
   OPENAI_KEY="$(cat /run/secrets/openai-api-key 2>/dev/null || true)"
   GITHUB_TOKEN="$(cat /run/secrets/github-pat 2>/dev/null || true)"
   ```

d. Build bwrap argument array. The EXACT mount order matters (broader ro-bind FIRST, specific rw-bind AFTER):

   **Namespace isolation:**
   - `--unshare-user --uid $(id -u) --gid $(id -g)` (user namespace with host UID/GID mapping)
   - `--unshare-ipc` (IPC isolation)
   - `--unshare-uts` (hostname isolation)
   - `--disable-userns` (prevent nested user namespace escape)
   - `--hostname "sandbox-$NAME"`
   - `--die-with-parent` (cleanup on parent exit)
   - `--new-session` (prevent terminal escape)

   **Nix store + daemon (read-only):**
   - `--ro-bind /nix/store /nix/store`
   - `--ro-bind /nix/var/nix/daemon-socket /nix/var/nix/daemon-socket`
   - `--ro-bind /nix/var/nix/db /nix/var/nix/db`
   - `--ro-bind /nix/var/nix/gcroots /nix/var/nix/gcroots`

   **NixOS profiles (read-only):**
   - `--ro-bind /run/current-system /run/current-system`
   - `--ro-bind-try /etc/profiles/per-user/dangirsh /etc/profiles/per-user/dangirsh`
   - `--ro-bind-try /home/dangirsh/.nix-profile /home/dangirsh/.nix-profile`

   **/etc selective (read-only):**
   - `/etc/resolv.conf`, `/etc/passwd`, `/etc/group`, `/etc/ssl`, `/etc/nix`, `/etc/static`
   - `--ro-bind-try` for: `/etc/hosts`, `/etc/nsswitch.conf`, `/etc/login.defs`, `/etc/subuid`, `/etc/subgid`, `/etc/containers`

   **Virtual filesystems:**
   - `--proc /proc` (shared PID namespace per decision)
   - `--dev /dev` (minimal synthetic devtmpfs)
   - `--tmpfs /tmp` with `--size 4294967296` (4GB tmpfs)

   **Project + sibling read-only (ORDER CRITICAL):**
   - `--ro-bind /data/projects /data/projects` FIRST
   - `--bind "$PROJECT_DIR" "$PROJECT_DIR"` AFTER (overrides for this specific project)

   **Curated home dotfiles (read-only):**
   - `--dir /home/dangirsh` (create home dir mountpoint)
   - `--ro-bind-try` for: `.gitconfig`, `.npmrc`, `.claude`, `.codex`, `.config/git`
   - `--bind-try /home/dangirsh/.local/share/containers /home/dangirsh/.local/share/containers` (rw for rootless Podman storage)

   **Podman support (rootless):**
   - `--bind-try /run/user/$(id -u)/containers /run/user/$(id -u)/containers`
   - `--ro-bind-try /run/user/$(id -u)/podman /run/user/$(id -u)/podman`

   **Environment (--clearenv + explicit --setenv):**
   - HOME=/home/dangirsh
   - USER=dangirsh
   - SHELL=/bin/bash
   - TERM=$TERM (from parent)
   - LANG=C.UTF-8
   - PATH="/run/current-system/sw/bin:/etc/profiles/per-user/dangirsh/bin:/home/dangirsh/.nix-profile/bin:/nix/var/nix/profiles/default/bin" (note: `docker` CLI is already available via Podman's dockerCompat — no sandbox-specific derivation needed)
   - SANDBOX=1
   - SANDBOX_NAME=$NAME
   - SANDBOX_PROJECT=$PROJECT_DIR
   - NIX_PATH (if needed for nix-shell)

   **API key injection (conditional):**
   - If ANTHROPIC_KEY non-empty: `--setenv ANTHROPIC_API_KEY "$ANTHROPIC_KEY"`
   - If OPENAI_KEY non-empty: `--setenv OPENAI_API_KEY "$OPENAI_KEY"`
   - If GITHUB_TOKEN non-empty: `--setenv GITHUB_TOKEN "$GITHUB_TOKEN"` and `--setenv GH_TOKEN "$GITHUB_TOKEN"`

   **chdir:**
   - `--chdir "$PROJECT_DIR"`

**2. Add Podman NixOS config to agent-compute.nix:**

```nix

  enable = true;
  dockerCompat = true;  # Locked decision from CONTEXT.md

environment.systemPackages = [

  # ... existing packages

```

**DX trade-off note (add as code comment):** With `dockerCompat = true`, the `docker` CLI binary in PATH resolves to Podman system-wide. The host Docker daemon (`virtualisation.docker.enable = true` in `docker.nix`) still runs via `dockerd.service` systemd unit and is unaffected — container lifecycle, networking, and volumes all continue working. However, host `docker ps`, `docker logs`, etc. will now show Podman state, not Docker state. To manage host Docker containers, use `systemctl` or invoke `dockerd` directly, or use the full path to the Docker CLI binary. If this proves unworkable during deployment testing (Plan 11-02), the fallback is to set `dockerCompat = false` and create a sandbox-local PATH derivation with a `docker -> podman` symlink instead.

**Important implementation notes:**
- `writeShellApplication` automatically runs shellcheck -- ensure the script is shellcheck-clean.
- The `--tmpfs /tmp --size 4294967296` syntax: bwrap expects `--size` to follow `--tmpfs` immediately as the next option. Verify this parses correctly (some bwrap versions use `--tmpfs /tmp` followed by `--size BYTES` as separate args; 0.11.0 confirmed to support this pattern).

- Do NOT bind `~/.ssh/` (decision: completely invisible).
- Do NOT bind `/var/run/docker.sock` (decision: no Docker socket in sandbox).
