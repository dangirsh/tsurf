# Plan 66-02: Generic NixOS Module for Secret Placeholder Proxy

## Objective

Replace `modules/secret-proxy.nix` (hardcoded Phase 22 Python proxy for Anthropic/claw-swap)
with a generic, reusable NixOS module that:
- Accepts a `services.secretProxy.services.<name>` attribute set
- For each declared service, generates a per-service systemd unit, dedicated system user,
  a TOML config file (for the Rust binary from Plan 66-01), and assertions preventing
  port collisions and public exposure
- Exposes a `config.services.secretProxy.services.<name>.bwrapArgs` output — a list of
  `["--setenv" "VAR" "VALUE"]` items ready to splice into a bwrap invocation

This plan rewrites the module only. The private overlay consumer (claw-swap config) and
migration of the old hardcoded service are handled in Plan 66-03.

IMPORTANT: This plan must NOT break `nix flake check` at any intermediate stage. The module
is written to be self-contained. The old `modules/secret-proxy.nix` is rewritten in-place —
no imports change in `hosts/services/default.nix` unless currently broken.

## must_haves

- `modules/secret-proxy.nix` defines `options.services.secretProxy.services` as an `attrsOf` submodule
- Each service sub-option has: `port` (int, required), `placeholder` (string or null, optional),
  `secrets` (attrsOf submodule with `headerName`, `secretFile`, `allowedDomains`),
  and read-only `bwrapArgs` output
- `placeholder` defaults to `"sk-placeholder-<name>"` when null
- `allowedDomains` is required (no default); missing it causes a NixOS eval error (no exceptions)
- Per-service systemd unit named `secret-proxy-<name>` with full hardening attrs
  (copy hardening block verbatim from Phase 22 module — `ProtectSystem = "strict"`, etc.;
  add `MemoryDenyWriteExecute = true` since Rust binary is safe to restrict)
- Per-service dedicated system user `secret-proxy-<name>` in group `secret-proxy`
  (shared group, per-service user for process isolation)
- TOML config file generated into `/run/secret-proxy/<name>.toml` via `systemd.tmpfiles.rules`
  or written by the service's `ExecStartPre` — choose tmpfiles for simplicity
- Assertion: `services.secretProxy.services.<name>.port` must not appear in
  `networking.firewall.allowedTCPPorts` — enforced via an eval-time assertion that no two
  services share a port. `internalOnlyPorts` in `networking.nix` is unnecessary for
  secret-proxy because the module hard-codes `127.0.0.1` binding; a 127.0.0.1-only listener
  cannot be reached from the public internet regardless of firewall rules
- Assertion: no two services may declare the same port (eval-time check)
- `bwrapArgs` output returns `["--setenv" "ANTHROPIC_BASE_URL" "http://127.0.0.1:${port}"]`
  for Anthropic-style services; the module derives the correct env var key from the
  declared secrets' header names (see implementation note below)
- `nix flake check` passes with the new module

## Implementation Notes

### bwrapArgs env var derivation

The module cannot know which env var the agent uses (`ANTHROPIC_BASE_URL`,
`OPENAI_BASE_URL`, etc.) unless told. Two approaches:

**Approach A (simpler):** Add an optional per-service `baseUrlEnvVar` option
(string, default `"ANTHROPIC_BASE_URL"`) that the user overrides for non-Anthropic services.
`bwrapArgs` then emits `["--setenv" "${baseUrlEnvVar}" "http://127.0.0.1:${toString port}"]`.

**Approach B (derive from header):** Map `x-api-key` → `ANTHROPIC_BASE_URL`,
`authorization` → `OPENAI_BASE_URL`, etc. Error-prone; breaks when header name doesn't
uniquely identify the provider.

**Use Approach A.** It is explicit and backward-compatible. The private overlay config
specifies `baseUrlEnvVar = "ANTHROPIC_BASE_URL"` for claw-swap.

### TOML config file location

Write the config to `/run/secret-proxy/<name>.toml` (tmpfs, recreated at boot).
Use `systemd.tmpfiles.rules` to create the directory, and write the TOML content
using `pkgs.writeText "secret-proxy-<name>.toml" "..."` stored in the Nix store,
then symlink or copy into `/run/` via `ExecStartPre`.

Simpler alternative: pass config as a Nix store path directly to `ExecStart`:
```nix
ExecStart = "${secretProxyBin}/bin/secret-proxy --config ${configFile}";
```
where `configFile = pkgs.writeText "secret-proxy-${name}.toml" (tomlContent)`.
This is cleaner — no tmpfiles needed for the config (secret file paths in the TOML
still point to sops-nix-managed `/run/secrets/...` paths).

**Use the `pkgs.writeText` store path approach** for the config file. The TOML is not
secret itself (it contains only file paths and header names, not key values). The actual
secret values stay in sops-nix-managed files referenced by path in the TOML.

### systemd unit structure per service

```nix
systemd.services."secret-proxy-${name}" = {
  description = "Secret placeholder proxy for ${name}";
  after = [ "network.target" "sops-nix.service" ];
  wantedBy = [ "multi-user.target" ];
  serviceConfig = {
    ExecStart = "${pkgs.secret-proxy}/bin/secret-proxy --config ${configFile}";
    User = "secret-proxy-${name}";
    Restart = "on-failure";
    RestartSec = "5s";
    # Full hardening (from Phase 22, plus MemoryDenyWriteExecute for Rust binary)
    NoNewPrivileges = true;
    ProtectSystem = "strict";
    ProtectHome = true;
    PrivateTmp = true;
    MemoryDenyWriteExecute = true;
    ProtectKernelTunables = true;
    ProtectKernelModules = true;
    ProtectKernelLogs = true;
    ProtectControlGroups = true;
    RestrictAddressFamilies = [ "AF_INET" "AF_INET6" ];
    RestrictNamespaces = true;
    RestrictRealtime = true;
    RestrictSUIDSGID = true;
    CapabilityBoundingSet = "";
    SystemCallArchitectures = "native";
    LockPersonality = true;
    PrivateDevices = true;
  };
};
```

### Module referencing the Rust binary

The module needs to reference `pkgs.secret-proxy`. Since `packages/secret-proxy.nix` is
not in `environment.systemPackages` (it's a module-internal tool), use:

```nix
let
  secretProxyPkg = pkgs.callPackage ../packages/secret-proxy.nix {};
in
```

or add it to the flake's overlay. The simplest approach: `pkgs.callPackage` with a
relative path from `modules/secret-proxy.nix` to `packages/secret-proxy.nix`.

      ```nix
      # modules/secret-proxy.nix
      # @decision PROXY-66-01: Generic secret placeholder proxy module.
      #   Replaces the Phase 22 hardcoded Python proxy with a backend-agnostic Rust
      #   binary + declarative NixOS interface. Per-service isolation: each service
      #   gets its own systemd unit, user, and port.
      # @decision PROXY-66-02: BASE_URL plain-HTTP approach (not CONNECT/TLS MITM).
      #   Agent sets ANTHROPIC_BASE_URL=http://127.0.0.1:<port>; proxy controls
      #   upstream destination. No CA certificate distribution needed.
      #   TLS SNI validation is unnecessary with this approach: the proxy sets the
      #   upstream URL directly from its config, so the agent cannot influence the
      #   upstream target via SNI or Host header manipulation.
      # @decision PROXY-66-03: File path interface for secrets — backend-agnostic.
      #   Module accepts secretFile paths (sops-nix, agenix, or any file provider).
      #   Real keys never appear in Nix store, env vars, or CLI args.
      { config, lib, pkgs, ... }:

      let
        secretProxyPkg = pkgs.callPackage ../packages/secret-proxy.nix {};

        serviceOpts = { name, config, ... }: {
          options = {
            port = lib.mkOption {
              type = lib.types.port;
              description = "Port the proxy listens on (127.0.0.1 only).";
            };
            placeholder = lib.mkOption {
              type = lib.types.str;
              default = "sk-placeholder-${name}";
              description = "Placeholder token injected into agent env. Mimic real key format.";
            };
            baseUrlEnvVar = lib.mkOption {
              type = lib.types.str;
              default = "ANTHROPIC_BASE_URL";
              description = "Env var set in bwrapArgs pointing to this proxy's base URL.";
            };
            secrets = lib.mkOption {
              type = lib.types.attrsOf (lib.types.submodule secretDef);
              default = {};
              description = "Secrets to inject, keyed by logical name.";
            };
            bwrapArgs = lib.mkOption {
              type = lib.types.listOf lib.types.str;
              readOnly = true;
              description = "bwrap args to splice into sandbox invocations (--setenv BASE_URL and --setenv PLACEHOLDER_VAR).";
            };
          };
          config = {
            bwrapArgs = [
              "--setenv" config.baseUrlEnvVar "http://127.0.0.1:${toString config.port}"
            ];
          };
        };

        secretDef = { ... }: {
          options = {
            headerName = lib.mkOption {
              type = lib.types.str;
              example = "x-api-key";
              description = "Request header to inject the real key into.";
            };
            secretFile = lib.mkOption {
              type = lib.types.path;
              description = "Path to file containing the real secret value (e.g. config.sops.secrets.\"...\".path).";
            };
            allowedDomains = lib.mkOption {
              type = lib.types.nonEmptyListOf lib.types.str;
              description = "Exact domain names the proxy is allowed to forward to.";
            };
          };
        };

        mkToml = name: svcCfg:
          let
            secretEntries = lib.mapAttrsToList (secretName: s:
              ''
                [[secret]]
                header = "${s.headerName}"
                name = "${secretName}"
                file = "${s.secretFile}"
                allowed_domains = [${lib.concatMapStringsSep ", " (d: ''"${d}"'') s.allowedDomains}]
              ''
            ) svcCfg.secrets;
          in
          pkgs.writeText "secret-proxy-${name}.toml" ''
            port = ${toString svcCfg.port}
            placeholder = "${svcCfg.placeholder}"
            ${lib.concatStrings secretEntries}
          '';

        mkService = name: svcCfg:
          let
            configFile = mkToml name svcCfg;
          in {
            "secret-proxy-${name}" = {
              description = "Secret placeholder proxy for ${name}";
              after = [ "network.target" "sops-nix.service" ];
              wantedBy = [ "multi-user.target" ];
              serviceConfig = {
                ExecStart = "${secretProxyPkg}/bin/secret-proxy --config ${configFile}";
                User = "secret-proxy-${name}";
                Restart = "on-failure";
                RestartSec = "5s";
                # @decision SEC66-01: Full systemd hardening for secret proxy.
                # MemoryDenyWriteExecute added vs Phase 22: Rust binary is safe to restrict.
                NoNewPrivileges = true;
                ProtectSystem = "strict";
                ProtectHome = true;
                PrivateTmp = true;
                MemoryDenyWriteExecute = true;
                ProtectKernelTunables = true;
                ProtectKernelModules = true;
                ProtectKernelLogs = true;
                ProtectControlGroups = true;
                RestrictAddressFamilies = [ "AF_INET" "AF_INET6" ];
                RestrictNamespaces = true;
                RestrictRealtime = true;
                RestrictSUIDSGID = true;
                CapabilityBoundingSet = "";
                SystemCallArchitectures = "native";
                LockPersonality = true;
                PrivateDevices = true;
              };
            };
          };

        mkUser = name: _svcCfg: {
          "secret-proxy-${name}" = {
            isSystemUser = true;
            group = "secret-proxy";
          };
        };

      in {
        options.services.secretProxy.services = lib.mkOption {
          type = lib.types.attrsOf (lib.types.submodule serviceOpts);
          default = {};
          description = "Per-service secret placeholder proxy declarations.";
        };

        config = lib.mkIf (config.services.secretProxy.services != {}) {
          users.groups.secret-proxy = {};
          users.users = lib.mkMerge (lib.mapAttrsToList mkUser config.services.secretProxy.services);
          systemd.services = lib.mkMerge (lib.mapAttrsToList mkService config.services.secretProxy.services);

          assertions = let
            allPorts = lib.mapAttrsToList (n: s: { name = n; port = s.port; }) config.services.secretProxy.services;
            portCounts = builtins.foldl' (acc: x:
              acc // { "${toString x.port}" = (acc."${toString x.port}" or []) ++ [ x.name ]; }
            ) {} allPorts;
            duplicates = lib.filterAttrs (_: services: builtins.length services > 1) portCounts;
          in [
            {
              assertion = duplicates == {};
              message = "services.secretProxy: duplicate ports detected: ${
                lib.concatStringsSep ", " (lib.mapAttrsToList (port: services:
                  "port ${port} used by [${lib.concatStringsSep ", " services}]"
                ) duplicates)
              }";
            }
          ];
        };
      }
      ```

## Risks

### Module evaluates before packages/secret-proxy.nix exists (MEDIUM)
Plan 66-01 and 66-02 are both wave 1 (parallel), but 66-02 references `../packages/secret-proxy.nix`.
If 66-02 is executed before 66-01, eval will fail. The executor must run both plans in the
same wave and ensure Plan 66-01's file creation happens before 66-02's eval check.
Mitigation: in the task description, note that Task 66-02-A should only run after
Task 66-01-E creates `packages/secret-proxy.nix`.

### Old sops template still referenced in private overlay (LOW)
The private overlay may reference `config.sops.templates."secret-proxy-env".path` in its
claw-swap configuration. This breaks after this plan removes the template. Mitigation:
Plan 66-03 handles private overlay migration (removes the old consumer, adds new config).
Until Plan 66-03 deploys, the private overlay must not be deployed.

### lib.types.port not available (LOW)
`lib.types.port` is available in NixOS 25.11. Use `lib.types.int` with an assertion
`value > 0 && value < 65536` as fallback if it causes an eval error.
