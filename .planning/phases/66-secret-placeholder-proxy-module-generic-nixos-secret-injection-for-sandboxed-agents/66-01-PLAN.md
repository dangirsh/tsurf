## Objective

Implement the `secret-proxy` Rust binary that replaces the Phase 22 hardcoded Python proxy.
The binary receives plain HTTP from sandboxed agents, validates the `Host` header against a
per-secret domain allowlist, replaces the placeholder token in the configured request header
with the real secret read from a file on disk, and forwards the request via HTTPS to the
hardcoded upstream. Single per-service binary instance; one port per declared service (per
the NixOS module schema); config via a TOML file generated by the NixOS module.

Architecture (BASE_URL plain-HTTP approach — research finding Q3/Q4):
- Incoming: plain HTTP on `127.0.0.1:<port>` (agent sets ANTHROPIC_BASE_URL=http://127.0.0.1:<port>)
- Validation: `Host` header must exactly match one of `allowed_domains` for the triggered secret
- Injection: replace the full value of the configured request header (`x-api-key`, `Authorization`, etc.)
  with the real secret loaded from `secret_file` at startup
- Forwarding: HTTPS to `upstream_host` (from `allowed_domains[0]`) via `reqwest`
- Rejection: 403 + log for any request where Host does not match allowlist

## must_haves

- `packages/secret-proxy/Cargo.toml` exists with `axum`, `reqwest`, `tokio`, `serde`, `toml` dependencies
- `packages/secret-proxy/src/main.rs` compiles cleanly with `cargo build`
- Binary accepts a single `--config <path>` argument pointing to a TOML config file
- Config TOML supports: `port` (u16), `placeholder` (String), and `[[secret]]` array
  - Each `[[secret]]` has: `header` (String), `name` (String), `file` (String, path), `allowed_domains` (Vec<String>)
- At startup: binary reads each `secret.file`, validates it is non-empty, caches in memory
- On each request: extracts `Host` header, checks against `allowed_domains` of all configured secrets
  - If no match: returns HTTP 403, logs `DENY: host=<X> not in allowlist`
  - If match: replaces placeholder in configured header with real key, forwards HTTPS to upstream
- Upstream forwarding uses `reqwest` with full TLS validation (no `danger_accept_invalid_certs`)
- Streaming response: streams upstream response body back to client (no full buffering)
- Skip headers list for forwarding (same as Phase 22): `x-api-key`, `authorization`, `host`,
  `content-length`, `transfer-encoding` on request; `transfer-encoding`, `connection` on response
- `packages/secret-proxy.nix` builds the binary via `rustPlatform.buildRustPackage` with correct `cargoHash`
- `flake.nix` exposes `packages.x86_64-linux.secret-proxy` so it can be referenced as `pkgs.secret-proxy`
  (via overlay or direct package attr — choose whichever is simpler given existing flake structure)
- `nix build .#secret-proxy` succeeds (or `nix build .#packages.x86_64-linux.secret-proxy`)

    <title>Create packages/secret-proxy/src/proxy.rs</title>
    <description>
      Create `packages/secret-proxy/src/proxy.rs` implementing the core request handler:

      Key logic:
      1. Extract `Host` header from incoming request (strip port suffix if present)
      2. Walk through all `LoadedSecret` entries; find one whose `allowed_domains` contains exact `host` match
      3. If no match: return `(StatusCode::FORBIDDEN, "Access denied: host not in allowlist\n")`
         and log `DENY host={host} secret_name={name}`
      4. If match: rebuild request headers — copy all incoming headers EXCEPT skip list
         (`x-api-key`, `authorization`, `host`, `content-length`, `transfer-encoding`)
         then set the secret's header to the cached real key value
      5. Forward via `reqwest` to `https://{host}{path_and_query}` with the rebuilt headers
      6. Stream the upstream response back (status code, headers minus skip list, body)

      Upstream host for forwarding: use `allowed_domains[0]` from the matched secret
      (the BASE_URL approach means the proxy, not the agent, controls the upstream destination).

      `LoadedSecret` struct holds: header name, real key value (String, loaded from file), allowed_domains.

      Skip list for request forwarding: `["x-api-key", "authorization", "host", "content-length",
        "transfer-encoding"]` (lowercase comparison).

      Skip list for response forwarding: `["transfer-encoding", "connection"]` (lowercase comparison).

    <description>
      Create `packages/secret-proxy/src/main.rs`:

      ```rust
      mod config;
      mod proxy;

      use std::{env, fs, net::SocketAddr, sync::Arc};
      use tracing_subscriber::EnvFilter;

      #[tokio::main]
      async fn main() {
          tracing_subscriber::fmt()
              .with_env_filter(EnvFilter::from_default_env()
                  .add_directive("secret_proxy=info".parse().unwrap()))
              .init();

          let args: Vec<String> = env::args().collect();
          let config_path = args.windows(2)
              .find(|w| w[0] == "--config")
              .map(|w| w[1].clone())
              .unwrap_or_else(|| {
                  eprintln!("Usage: secret-proxy --config <path>");
                  std::process::exit(1);
              });

          let config_str = fs::read_to_string(&config_path)
              .unwrap_or_else(|e| { eprintln!("Cannot read config {config_path}: {e}"); std::process::exit(1); });
          let config: config::Config = toml::from_str(&config_str)
              .unwrap_or_else(|e| { eprintln!("Invalid config: {e}"); std::process::exit(1); });

          // Load secrets from files at startup; fail fast if any file is unreadable/empty
          let loaded: Vec<proxy::LoadedSecret> = config.secret.iter().map(|s| {
              let key = fs::read_to_string(&s.file)
                  .unwrap_or_else(|e| { eprintln!("Cannot read secret '{}' from {}: {}", s.name, s.file, e); std::process::exit(1); });
              let key = key.trim().to_string();
              if key.is_empty() {
                  eprintln!("Secret '{}' is empty (file: {})", s.name, s.file);
                  std::process::exit(1);
              }
              proxy::LoadedSecret {
                  header: s.header.clone(),
                  key,
                  allowed_domains: s.allowed_domains.clone(),
              }
          }).collect();

          let placeholder = config.placeholder.clone();
          let state = Arc::new(proxy::ProxyState { placeholder, secrets: loaded });
          let app = proxy::make_router(state);

          let addr = SocketAddr::from(([127, 0, 0, 1], config.port));
          tracing::info!("secret-proxy listening on {addr}");
          let listener = tokio::net::TcpListener::bind(addr).await.unwrap();
          axum::serve(listener, app).await.unwrap();
      }
      ```

## Risks

### cargoHash vs cargoLock.lockFile confusion (LOW)
`cargoLock.lockFile` is cleaner for in-tree crates because it doesn't require recomputing a hash when dependencies change. If the evaluator has trouble with it, fall back to `cargoHash` using the `nix build` hash mismatch output.

### axum API churn (LOW)
axum 0.8 released 2025-02. The request handler pattern changed slightly from 0.7. Use the `axum::extract::{Request, State}` pattern; avoid old `hyper::Request` type aliases. If 0.8 is not in nixpkgs 25.11 yet, pin to 0.7 (adjust Cargo.toml accordingly).

### reqwest streaming (MEDIUM)
Streaming the upstream response back requires `bytes_stream()` from reqwest and an `axum::body::Body::from_stream()` on the response. If streaming causes lifetime/Send issues, fall back to full-buffer approach (collect body then send) — this is acceptable for the MVP since AI API responses are bounded in size.
