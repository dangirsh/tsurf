# Plan 39-02: Wire conway-dashboard into neurosys

## Goal

Add `dangirsh/conway-dashboard` as a `flake = false` input to neurosys,
create `modules/automaton-dashboard.nix` that references the external repo
for its Python script and HTML, and wire it into the homepage + networking.

## Decisions Made

- **DASH-10:** `pkgs.writeShellScriptBin` wraps `python3 ${inputs.conway-dashboard}/server.py`.
  The Python file is in the Nix store (via the flake input) so it's immutable
  at runtime. No flake8 constraints imposed.
- **DASH-11:** `DASHBOARD_HTML_PATH` and `AUTOMATON_DB_PATH` injected via
  systemd `Environment=` pointing into the Nix store and the state dir.
  No secrets needed for this service.

## must_haves

1. `flake.nix` declares `conway-dashboard` input: `github:dangirsh/conway-dashboard`, `flake = false`
2. `modules/automaton-dashboard.nix` exists with `@decision` annotations DASH-01 through DASH-10
3. Systemd service `automaton-dashboard` defined, running as `automaton` user
4. `ExecStart` invokes `python3 ${inputs.conway-dashboard}/server.py` (via `writeShellScriptBin`)
5. `DASHBOARD_HTML_PATH` points to `${inputs.conway-dashboard}/dashboard.html`
6. `automaton` user added to `systemd-journal` group
7. Port 9093 in `internalOnlyPorts` in `modules/networking.nix`
8. `modules/default.nix` imports `./automaton-dashboard.nix`
9. `modules/homepage.nix` has "Conway Automaton" entry (Infrastructure section)
10. `modules/repos.nix` clones `dangirsh/conway-dashboard` to `/data/projects/conway-dashboard`
11. `nix flake check` passes for all configurations (neurosys + ovh)
12. Committed and pushed

    <title>Create modules/automaton-dashboard.nix</title>
    <description>
      Create `/data/projects/neurosys/modules/automaton-dashboard.nix`.

      **Module header with @decision annotations:**
      ```nix
      # modules/automaton-dashboard.nix
      # Conway Automaton monitoring dashboard — read-only web UI on port 9093
      #
      # @decision DASH-01: Separate module (one-concern-per-module convention)
      # @rationale: automaton.nix is 257+ lines; dashboard is distinct concern.
      #
      # @decision DASH-02: Code in dangirsh/conway-dashboard private repo
      # @rationale: Keeps Conway-specific app code versioned independently from
      #   NixOS config. flake = false input (same pattern as automaton input).
      #
      # @decision DASH-03: pkgs.writeShellScriptBin wraps python3 from store
      # @rationale: Avoids writePython3Bin flake8 line-length enforcement.
      #   Python code stays readable in the external repo.
      #
      # @decision DASH-04: Runs as automaton user (owns state.db, no new user)
      # @rationale: Simpler than a separate user + ACL. ReadOnlyPaths +
      #   SQLite ?mode=ro provide defense-in-depth.
      #
      # @decision DASH-05: Port 9093 (monitoring range, Tailscale-only)
      # @rationale: Adjacent to Prometheus (9090), secret-proxy (9091).
      #   Not used by any deployed service. See NET-07 assertion.
      ```

      **Module arguments:** `{ config, pkgs, inputs, lib, ... }:`

      **let block:**
      ```nix
      let
        dashboardBin = pkgs.writeShellScriptBin "automaton-dashboard" ''
          exec ${pkgs.python3}/bin/python3 \
            ${inputs.conway-dashboard}/server.py
        '';
      in
      ```

      **Systemd service:**
      ```nix
      {
        systemd.services.automaton-dashboard = {
          description = "Conway Automaton monitoring dashboard";
          after = [
            "network-online.target"
            "conway-automaton.service"
          ];
          wants = [ "network-online.target" ];
          wantedBy = [ "multi-user.target" ];

          serviceConfig = {
            ExecStart = "${dashboardBin}/bin/automaton-dashboard";
            User = "automaton";
            Group = "automaton";
            Environment = [
              "DASHBOARD_HTML_PATH=${inputs.conway-dashboard}/dashboard.html"
              "AUTOMATON_DB_PATH=/var/lib/automaton/.automaton/state.db"
            ];
            Restart = "on-failure";
            RestartSec = "10s";

            NoNewPrivileges = true;
            ProtectSystem = "strict";
            ReadOnlyPaths = [ "/var/lib/automaton" ];
          };
        };

        users.users.automaton.extraGroups = [ "systemd-journal" ];
      }
      ```

      The `After = conway-automaton.service` is ordering-only — it does not
      require the automaton to exist. Both `neurosys` and `ovh` hosts evaluate
      cleanly (systemd ignores `After=` for non-existent units).
    </description>

## Risks and Mitigations

### Private repo auth for flake fetch (LOW)
Nix needs to fetch `github:dangirsh/conway-dashboard` during build. If the
build machine has `~/.config/nix/nix.conf` with `access-tokens` for GitHub,
this works automatically. If not, `nix flake update` may prompt for credentials.
On neurosys, the `github-pat` sops secret is available but the Nix daemon uses
system credentials. Mitigation: flake.lock pins the exact commit — remote fetch
only happens when locking or updating, not on each rebuild.

### OVH host compatibility (LOW)
If `hosts/ovh/default.nix` imports `./modules` (all modules), it will include
`automaton-dashboard.nix`. The `automaton` user may not exist on OVH, causing
`extraGroups` to fail evaluation. Check `hosts/ovh/default.nix` — if it does
not import `automaton.nix`, also skip `automaton-dashboard.nix`.
Mitigation: add the OVH host check to task 39-02-D.
