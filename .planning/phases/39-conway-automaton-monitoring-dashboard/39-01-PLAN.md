# Plan 39-01: Create dangirsh/conway-dashboard private repo

## Goal

Create a private GitHub repo `dangirsh/conway-dashboard` containing the Python
HTTP server (`server.py`) and HTML dashboard (`dashboard.html`). This repo
becomes a flake input to neurosys (plan 39-02), keeping Conway-specific
dashboard code separate from the NixOS configuration.

## Decisions Made

- **DASH-08:** Dashboard code lives in `dangirsh/conway-dashboard` (private),
  not inline in neurosys. Consistent with how other services are tracked
  (`dangirsh/parts`, `dangirsh/claw-swap`).
- **DASH-09:** `flake = false` input (like `automaton`). No flake wrapping
  needed — neurosys just needs the source files.
- **DASH-10:** `pkgs.writeShellScriptBin` wraps `python3 /nix/store/…/server.py`
  rather than `pkgs.writers.writePython3Bin`. The Python code lives in the
  external repo without the 79-char flake8 constraint, making it readable.

## must_haves

1. `dangirsh/conway-dashboard` private repo exists on GitHub
2. Repo contains `server.py` — Python HTTP server on port 9093
3. Repo contains `dashboard.html` — self-contained HTML/CSS/JS dashboard
4. `server.py` serves `GET /` (HTML) and `GET /api/status` (JSON)
5. `/api/status` returns all required fields: agent_state, current_tier,
   credits_cents, total_turns, active_goal, task_counts, recent_tool_calls,
   hourly_spend_cents, journal_logs
6. `dashboard.html` auto-refreshes every 5s via `fetch('/api/status')`
7. Both files committed and pushed to the repo's default branch

    <title>Create server.py</title>
    <description>
      Create `/data/projects/conway-dashboard/server.py`.

      The Python server uses only stdlib — no pip dependencies. It serves two
      routes and reads from the automaton SQLite state.db + journald.

      **Constants (at module level):**
      ```python
      import http.server
      import json
      import os
      import sqlite3
      import subprocess
      import socketserver

      PORT = 9093
      DB_PATH = os.environ.get(
          "AUTOMATON_DB_PATH",
          "/var/lib/automaton/.automaton/state.db"
      )
      HTML_PATH = os.environ.get("DASHBOARD_HTML_PATH", "dashboard.html")
      JOURNAL_UNIT = "conway-automaton"
      ```

      **`get_status()` function:**
      - Opens DB: `sqlite3.connect(f"file:{DB_PATH}?mode=ro", uri=True, timeout=5)`
        - First check `os.path.exists(DB_PATH)`, return error dict if missing
      - Queries to run (each in its own try/except for graceful degradation):
        - `kv` table: `SELECT value FROM kv WHERE key = ?` for keys
          `agent_state`, `financial_state` (JSON string with creditsCents),
          `current_tier`
        - `turns`: `SELECT COUNT(*) FROM turns`
        - `inference_costs`:
          `SELECT COALESCE(SUM(cost_cents), 0) FROM inference_costs WHERE created_at > datetime('now', '-1 hour')`
        - `goals`: `SELECT id, title, status FROM goals WHERE status = 'active' LIMIT 1`
        - If active goal found, `task_graph`:
          `SELECT status, COUNT(*) FROM task_graph WHERE goal_id = ? GROUP BY status`
        - `tool_calls` joined with `turns` for recent 20:
          `SELECT tc.name, tc.duration_ms, t.timestamp FROM tool_calls tc JOIN turns t ON tc.turn_id = t.id ORDER BY t.timestamp DESC LIMIT 20`
        - `identity`: `SELECT value FROM identity WHERE key = 'name'`
      - Parse `financial_state` JSON: extract `creditsCents`, `usdcBalance`
      - Returns a dict with all fields; missing/failed fields default to None

      **`get_journal_logs()` function:**
      - `subprocess.run(["journalctl", "-u", JOURNAL_UNIT, "--output=json",
          "--no-pager", "-n", "30", "--reverse"], capture_output=True,
          text=True, timeout=5)`
      - Parse each line as JSON, extract `MESSAGE` and
        `__REALTIME_TIMESTAMP` (microseconds → ISO 8601)
      - Return list of `{"message": str, "timestamp": str}` dicts
      - Catch all exceptions, return `[]` on any failure

      **HTTP handler class (`Handler(http.server.BaseHTTPRequestHandler)`):**
      - `do_GET`:
        - `"/"`: read HTML_PATH, send 200 with `Content-Type: text/html`
        - `"/api/status"`: merge `get_status()` and `get_journal_logs()` into
          one dict, send 200 with `Content-Type: application/json`
        - Otherwise: 404
      - Override `log_message` to write to stderr (systemd captures it)

      **Main block:**
      ```python
      if __name__ == "__main__":
          socketserver.TCPServer.allow_reuse_address = True
          with socketserver.TCPServer(("", PORT), Handler) as server:
              print(f"Serving on port {PORT}", flush=True)
              server.serve_forever()
      ```

      (The `allow_reuse_address = True` is critical for service restarts —
      without it, EADDRINUSE kills the service during NixOS activation.)
