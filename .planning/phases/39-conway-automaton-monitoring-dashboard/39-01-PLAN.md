# Plan 39-01: Conway Automaton Monitoring Dashboard

## Goal

Build a lightweight, Tailscale-only web dashboard on port 9093 that displays
live Conway Automaton status by reading from the SQLite state.db and journald.
The dashboard shows: agent state, survival tier, credits balance, total turns,
current goal + task progress, recent tool calls, spend rate ($/hr), and recent
journal logs. Auto-refreshes every 5 seconds via vanilla JS fetch. Linked from
the main homepage dashboard. No new secrets, no external API calls.

## Decisions Made (from Research)

- **DASH-01:** Python HTTP server (`http.server` stdlib) matches the existing
  secret-proxy.nix pattern. No new runtime dependencies. Use
  `pkgs.writers.writePython3Bin` for the server (flake8-enforced, 79 char
  line limit) and `pkgs.writeText` for the HTML file.
- **DASH-02:** Read-only SQLite access via `?mode=ro` URI parameter. Runs as
  the `automaton` user for simplicity (already owns state.db). WAL mode allows
  concurrent readers.
- **DASH-03:** Port 9093 (monitoring range, after 9090 Prometheus and 9091
  secret-proxy). Not used by any deployed service.
- **DASH-04:** Separate module `automaton-dashboard.nix` per one-module-per-
  concern convention (`automaton.nix` is already 257 lines).
- **DASH-05:** Static HTML with vanilla JS auto-refresh (5s interval via
  `fetch('/api/status')`). No SPA framework.
- **DASH-06:** Journal access via `journalctl` subprocess, not the systemd-
  journal HTTP gateway. The `automaton` user is added to the `systemd-journal`
  group for read access.
- **DASH-07:** No external API calls. All data from local SQLite + journald.
  Credits balance is cached in the `kv` table by the automaton's heartbeat.

## must_haves

1. `modules/automaton-dashboard.nix` exists with a Python HTTP server serving `GET /` (HTML) and `GET /api/status` (JSON)
2. The Python server opens state.db in read-only mode (`?mode=ro`) and handles the case where the DB does not exist (graceful error)
3. The `/api/status` JSON response includes: agent_state, current_tier, credits (from `financial_state` KV), total turns, active goal + tasks, recent tool calls (last 20), hourly spend rate, and recent journal logs
4. The HTML page auto-refreshes every 5 seconds via `fetch('/api/status')` and renders the dashboard with no page reload
5. The systemd service runs as `User=automaton` with `After=conway-automaton.service`
6. Port 9093 is added to `internalOnlyPorts` in `modules/networking.nix`
7. `modules/default.nix` imports `./automaton-dashboard.nix`
8. `modules/homepage.nix` has a "Conway Automaton" entry linking to the dashboard
9. `nix flake check` passes for all configurations (neurosys + ovh)
10. The `automaton` user is added to the `systemd-journal` group for journalctl access

            - `HTML_PATH` read from env `DASHBOARD_HTML_PATH`
            - `JOURNAL_UNIT = "conway-automaton"`

         c. Define a `get_status()` function that:
            - Opens the SQLite DB in read-only mode:
              `sqlite3.connect(f"file:{DB_PATH}?mode=ro", uri=True, timeout=5)`
            - Queries `kv` table for `agent_state`, `financial_state` (JSON),
              `current_tier`
            - Queries `turns` for `COUNT(*)` (total turns)
            - Queries `inference_costs` for hourly spend:
              `SELECT SUM(cost_cents) FROM inference_costs WHERE created_at > datetime('now', '-1 hour')`
            - Queries `goals` for the active goal:
              `SELECT id, title, status FROM goals WHERE status = 'active' LIMIT 1`
            - If an active goal exists, queries `task_graph` for task counts:
              `SELECT status, COUNT(*) FROM task_graph WHERE goal_id = ? GROUP BY status`
            - Queries `tool_calls` joined with `turns` for recent calls:
              `SELECT tc.name, tc.duration_ms, t.timestamp FROM tool_calls tc JOIN turns t ON tc.turn_id = t.id ORDER BY t.timestamp DESC LIMIT 20`
            - Queries `identity` for agent name:
              `SELECT value FROM identity WHERE key = 'name'`
            - Catches `sqlite3.OperationalError` (table doesn't exist, DB
              locked, etc.) and returns partial/default data
            - Closes the connection in a `finally` block
            - Returns a dict with all the above data

         d. Define a `get_journal_logs()` function that:
            - Runs `subprocess.run(["journalctl", "-u", JOURNAL_UNIT, "--output=json", "--no-pager", "-n", "30", "--reverse"], capture_output=True, text=True, timeout=5)`
            - Parses each line as JSON, extracts `MESSAGE` and
              `__REALTIME_TIMESTAMP` (convert microseconds to ISO 8601)
            - Returns a list of `{"message": ..., "timestamp": ...}` dicts
            - Catches exceptions and returns an empty list

         e. Define the HTTP handler class extending
            `http.server.BaseHTTPRequestHandler`:
            - `do_GET(self)`:
              - If `self.path == "/"`: serve the HTML file (read from
                `HTML_PATH`, `Content-Type: text/html`)
              - If `self.path == "/api/status"`: call `get_status()` and
                `get_journal_logs()`, merge into one dict, return as JSON
                (`Content-Type: application/json`)
              - Otherwise: 404
            - Override `log_message` to write to stderr (systemd captures it)

         f. Main block:
            - `socketserver.TCPServer.allow_reuse_address = True`
              (needed for service restarts, per MEMORY.md secret-proxy lesson)
            - Create `socketserver.TCPServer(("", PORT), Handler)`
            - Print `f"Serving on port {PORT}"` to stderr
            - `server.serve_forever()`

         **IMPORTANT flake8 constraints** (enforced by `writePython3Bin`):
         - E501: All lines must be <= 79 characters. Break long strings with
           concatenation or parenthesized expressions.
         - E305: Two blank lines after class definitions before next function.
         - Use short variable names where needed to stay under 79 chars.
         - SQL queries can be assigned to variables to keep the execute() call
           short.

         **Error handling requirements:**
         - If the DB file doesn't exist: `get_status()` returns a dict with
           `"error": "Database not found"` and default/empty values for all
           fields. The HTML should display "Automaton not running".
         - If a table doesn't exist (schema version mismatch): catch
           `sqlite3.OperationalError` per-query and use defaults for that
           field. Other fields still populate.
         - If journalctl fails: return empty logs list, don't crash.

      ```nix
      systemd.services.automaton-dashboard = {
        description = "Conway Automaton monitoring dashboard";
        after = [
          "network-online.target"
          "conway-automaton.service"
        ];
        wants = [ "network-online.target" ];
        wantedBy = [ "multi-user.target" ];

        serviceConfig = {
          ExecStart = "${dashboardServer}/bin/automaton-dashboard";
          User = "automaton";
          Group = "automaton";
          Environment = [
            "DASHBOARD_HTML_PATH=${dashboardHtml}"
            "AUTOMATON_DB_PATH=/var/lib/automaton/.automaton/state.db"
          ];
          Restart = "on-failure";
          RestartSec = "10s";

          # Systemd hardening
          NoNewPrivileges = true;
          ProtectSystem = "strict";
          ReadOnlyPaths = [ "/var/lib/automaton" ];
        };
      };
      ```

      on top of the `?mode=ro` SQLite flag.

## Risks and Mitigations

### flake8 E501 line length in Python (MEDIUM)

`pkgs.writers.writePython3Bin` enforces 79-character line limits. SQL queries
and HTML path references may exceed this. Mitigation: assign long strings to
variables, use parenthesized line continuations, keep function names short.
This was successfully handled in the secret-proxy module.

### SQLite table schema mismatch (LOW)

If the automaton's schema evolves (currently version 10), queries may fail on
missing columns or tables. Mitigation: each query is wrapped in try/except
`sqlite3.OperationalError`, returning default values for that field. The
dashboard degrades gracefully — some panels show "N/A" instead of crashing.

### state.db does not exist yet (MEDIUM)

If the automaton has never been started, `/var/lib/automaton/.automaton/state.db`
will not exist. Mitigation: `get_status()` checks `os.path.exists(DB_PATH)`
first and returns `{"error": "Database not found"}`. The HTML renders
"Automaton not running" in this case.

### NixOS option merge for extraGroups (LOW)

The `automaton` user is defined in `modules/automaton.nix` with `isSystemUser`
and `group = "automaton"`. Adding `extraGroups = ["systemd-journal"]` in the
dashboard module uses NixOS option merging (lists are concatenated). This is
safe and well-tested NixOS behavior.

### journalctl subprocess timeout (LOW)

If journald is slow, the `subprocess.run` call has a 5-second timeout. If it
exceeds that, the dashboard returns empty logs. The HTTP request itself is not
blocked — only the journal section is empty.

## Notes for Implementer

- **Python line length**: The hardest part of this task is keeping all Python
  code under 79 characters while maintaining readability. Strategies:
  - Assign SQL queries to named variables before `cursor.execute()`
  - Use short variable names for frequently-used objects (`c` for cursor,
    `r` for result, `d` for data dict)
  - Break f-strings across multiple lines with parentheses
  - The HTML is in a separate `pkgs.writeText` derivation, so it has no
    line length constraint

- **HTML is unconstrained**: Unlike the Python, the HTML/CSS/JS in
  `pkgs.writeText` has no linter. Take advantage of this for readability.
  Use inline `<style>` and `<script>` blocks. No external resources (the
  dashboard must work without internet access).

- **Test locally**: After `nix flake check` passes, you can test the Python
  server locally (without deploying) by:
  ```bash
  nix build .#nixosConfigurations.neurosys.config.systemd.services.automaton-dashboard
  ```
  (This only evaluates, it does not build a runnable server. Full testing
  requires deploy.)

- **`allow_reuse_address`**: Set `socketserver.TCPServer.allow_reuse_address = True`
  before creating the server instance. Without this, the service fails to
  restart with EADDRINUSE during NixOS activation (learned from secret-proxy,
  per MEMORY.md).

- **Financial state parsing**: The `financial_state` KV value is a JSON string:
  `{"creditsCents": N, "usdcBalance": N, "lastChecked": "ISO8601"}`. Parse
  with `json.loads()` and convert `creditsCents` to dollars for display
  (`creditsCents / 100`).

- **Spend rate**: Query `inference_costs` for the last hour:
  `SELECT SUM(cost_cents) FROM inference_costs WHERE created_at > datetime('now', '-1 hour')`.
  Convert to $/hr: `total_cents / 100`. Display as `$X.XX/hr`.

- **Tool calls display**: Show tool name and duration_ms (converted to
  seconds). Truncate long tool names if needed for display. The `turns`
  table has the timestamp — join on `turn_id`.
